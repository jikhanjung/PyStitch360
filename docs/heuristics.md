# 도메인 휴리스틱 카탈로그

검출·분류가 완벽하지 않은 현 단계에서 쓰는 **약한 규칙(축구 도메인
사전지식)** 의 목록. E2E 검출 품질이 올라가면 하나씩 불필요해질 수
있으므로, 새 규칙을 넣거나 폐기할 때 여기에 기록한다.

각 항목: 규칙 → 근거 → 구현 위치 → 주요 파라미터 → 폐기 조건.

## 공 (검출·트랙)

| 규칙 | 근거 | 위치 | 파라미터 |
|---|---|---|---|
| 정적+고립 공 기각 | 경기 중 공은 장시간 정지·고립하지 않음 (낙엽·마킹 오검출) | `ptz.build_plan` decoy 필터 | static 30px / iso 700px / 3s |
| 점프 게이팅 | 공은 프레임당 일정 거리 이상 못 움직임 | `ptz.link_ball_tracks` | 120px/frame |
| 속도 상한 = 검출 점프 | 실제 공 속도 ≤ ~45 m/s, 초과는 ID 점프 | `highlights.ball_speed_events` | 45 m/s |
| 강슛/롱패스 문턱 | 순변위 속도 20 m/s↑ 지속 = 의미 있는 킥 | 〃 | 20 m/s, 2스텝, 순변위 0.6× |
| 공중볼 탄도 | 공중 공의 지면 투영은 h/(h−z) 배율로 밀림 — 탄도 피팅으로 복원 | `airborne.detect_airborne_segments` | 0.5~3.5s, apex ≥0.8m, ≤0.85h |
| 갭필 주입 마커 | 저신뢰 주입 검출은 후보 행 길이 6 (6번째 = 원 conf) | `ptz.gapfill_analysis` / `export_training_labels` | conf 바닥 0.26 |

## 선수 (분류·역할)

| 규칙 | 근거 | 위치 | 파라미터 |
|---|---|---|---|
| 팀 = 유니폼 색 군집 | 두 팀 + 기타(3군집), 검출 수 상위 2개가 팀 | `ptz.classify_teams` | k=3, s-가중 색 벡터 |
| GK/심판만 색 전파 | 시드 전파는 "다른 색 옷" 전제 — 팀 시드는 자신만 (031 버그) | 〃 | 역할 ≥3 만 |
| **GK 단일성** | 팀당 GK 한 명 — 같은 GK 역할이 시간상 겹치는 트랙릿 여럿에 못 붙음. 비겹침 조각(ID 갈라짐)은 전파 유지 | 〃 (전파 후 필터) | 겹침 허용 0.5s |
| 심판 위치 통계 | 선심 = 사이드라인 ±3.5m 상주 + 터치라인 방향 이동, 주심 = 필드 내부 + 광범위 | `events.classify_referees` | 3.5m / x-span 8m / inside 75% |
| 선심 기 신호 포즈 | 올림 지속 = 파울, 올렸다 수평 지시 = 오프사이드 | `events.classify_flag_signal` | raise 0.75 / ext 0.6 / 0.8s |
| 선수 속도 상한 | 사람 달리기 ≤ ~11 m/s, 초과 스텝 = 트래커 점프 | `report.movement_stats` | 11 m/s |
| 지터 리샘플 | 검출 지터(±0.3m)는 랜덤워크로 거리를 부풀림 → 0.5s 창 평균 | 〃 | 0.5s bin |

## 트랙릿 병합 (ReID 라이트)

| 규칙 | 근거 | 위치 | 파라미터 |
|---|---|---|---|
| 시공간 연속 | 같은 사람의 조각은 끝↔시작이 사람 속도로 도달 가능 | `tracklets.suggest_links` | Δt<3s, 8m/s+2m |
| 시간 비겹침 | 같은 사람이 동시에 두 곳에 없음 | 〃 | 겹침 허용 0.3s |
| 유니폼 색 근접 | 같은 사람 = 같은 옷 | 〃 | 색 거리 60 |
| 역할 일치 | 다른 팀/역할 조각은 다른 사람 | 〃 | — |
| 체인 (후속/선행 1개) | 한 시점에 한 사람은 한 조각 | 〃 | — |

## 경기 이벤트

| 규칙 | 근거 | 위치 | 파라미터 |
|---|---|---|---|
| 킥오프 = 규칙 강제 배치 | 전원 자기 진영 + 센터서클 소수 + 직후 붕괴 (경기 규칙 8조) | `events.detect_kickoffs` | sep 0.8/0.55, circle ≤4 |
| **경기 중 킥오프 = 직전 골** | 득점 후에만 센터 재개 | `ptz_tab.suggest_goals` | −45s 이벤트, [−70,−5s] 구간 |
| 롱 휘슬 = 주요 판정 | 킥오프·파울·종료는 긴 호각 | `events.detect_kickoffs` / `highlights` | ≥0.8s |
| 공중볼 → 박스 = 슛/크로스 | 골문 쪽 착지 공중볼은 장면 가치 높음 | `highlights.airborne_box_events` | 끝 25m, 폭 +4m |
| 하이라이트 규칙 창 | 이벤트 종류별 앞뒤 여유 (P03-1 표) | `highlights.build_highlights` | 가중 user 5.0 > … |

## 유지 규칙

- 새 휴리스틱 추가/파라미터 변경/폐기 시 이 문서와 해당 devlog 에 기록.
- 각 규칙은 가능하면 tests/ 에 합성 테스트를 둔다 (GK 단일성:
  `test_teams.py`, 속도/지터: `test_report.py`, `test_highlights.py` 등).
- 실데이터 검증(P03-6)에서 오탐이 나오면 여기 파라미터부터 의심.
