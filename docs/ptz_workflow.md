# 가상 PTZ 워크플로우 — 분석부터 산출물까지

경기 영상 하나를 스티칭된 파노라마에서 하이라이트 클립·리포트까지
가져가는 전체 절차. 분석 메뉴 항목들은 순서 의존성이 있으므로 아래
순서를 따른다. (P03 로드맵 1~5 구현 기준, devlog 027~033)

## 진입점 (P05 분리)

| 실행 | 용도 |
|---|---|
| `python pitchstitch.py` | **PitchStitch** — 스티칭 전용 (탭 1~3, torch 불필요) |
| `python pitchwatch.py [pano.mp4]` | **PitchWatch** — 경기 분석 전용 (이 문서의 본문) |
| `python main.py` | 통합 실행 (전환기 호환 — 두 기능 모두) |

PitchStitch 내보내기 완료 대화의 "PitchWatch 에서 열기" 버튼이 두 앱을
잇는다 (통합 실행에서는 "PTZ 탭에서 열기").

## 0. 준비

| 파일 | 내용 | 만든 주체 |
|---|---|---|
| `<이름>.mp4` | 스티칭된 파노라마 | 내보내기 탭 |
| `<이름>.analysis.json` | 순수 분석 (검출) — 검수로 불변 | 자동 분석 |
| `<이름>.ptz.json` | 사용자 편집 (키프레임·역할·병합·경기정보 등) | GUI 검수 |
| `<이름>.events.json` | 이벤트 (킥오프·공중볼 캐시·하이라이트) | 분석 메뉴 |
| `<이름>.whistle.json` | 호각 트랙 | scripts/whistle.py |
| `<이름>.scrub.mp4` | 스크럽 프록시 (즉시 탐색) | GUI 버튼 |

## 1. 분석·검수 (기반)

1. **파노라마 열기** → 스크럽 프록시 생성 (권장).
2. **자동 공/선수 분석** (yolo11m 권장) — `.analysis.json` 캐시.
3. **공 트랙 검수**: 타임라인에서 무시(빨강)/승격(자홍)/키프레임 편집.
   필요 시 분석 메뉴 **갭필 2차 패스**, 우클릭 시드 전파.
4. **경기장 캘리브레이션**: 경기장 탭에서 랜드마크 찍기 (+실측 크기
   입력). 이후 단계 대부분(킥오프·병합·리포트·공중볼)이 필드 좌표 의존.
5. **선수 역할 검수**: 팀 오분류 수정, GK/심판 지정.
   팀 수정은 그 선수에게만 적용된다 (devlog 031).

## 2. 이벤트 (오디오 → 킥오프 → 심판)

6. `python scripts/whistle.py <pano.mp4>` — 호각 트랙 추출 (~20초).
7. 분석 메뉴 **킥오프 검출 (호각 × 대형)** — `.events.json`.
8. (선택) `python scripts/referee.py <pano.mp4>` — 심판 분류 + 선심
   기 신호 (파울/오프사이드) → 하이라이트 재료.

## 3. 트랙릿 병합 (devlog 030)

9. 분석 메뉴 **트랙릿 병합 제안** — 선수 목록에 `#tid (+N)` 그룹.
   우클릭: 선택 병합 / 멤버 분리 / 그룹 해체. 전부 비파괴 — 팀 분류가
   틀린 조각도 일단 병합하면 대표 기준으로 표시가 바로잡힌다.

## 4. 경기 정보·득점 (devlog 029)

10. 분석 메뉴 **경기 정보** — 전/후반, 하프 길이(예: 30분), 킥오프
    앵커. 중단 구간(hydration break)은 타임라인에서 IN/OUT 마커 후
    우클릭 "IN/OUT → 경기 중단 구간" (시계 정지).
11. 분석 메뉴 **득점 역추론** — 경기 중 킥오프마다 '골?' 이벤트와
    '골 추정' 구간 제안 → 구간 재생으로 확인 후 이벤트 이름을
    **골1/골2** 로 변경 (시계 스코어 반영) 또는 삭제.

## 5. 하이라이트 (devlog 027)

12. 분석 메뉴 **하이라이트 후보 생성** — 킥오프·기 신호·롱 휘슬·
    공중볼·속도 급증·사용자 이벤트 융합 → 하이라이트 레인 (호박색).
13. 검수: 바 우클릭 → **구간 재생** / 수락 / 제외 / 경계 조정
    (IN/OUT 마커 재사용). 놓친 장면은 "여기에 하이라이트 추가" 또는
    "IN/OUT → 새 하이라이트" (수동 = 바로 수락, 재생성에도 보존).
14. 분석 메뉴 **하이라이트 일괄 내보내기** — 구간별
    `<이름>_h01_킥오프.mp4`. 시계/미니맵 옵션 동일 적용.

## 6. 내보내기·리포트

15. **PTZ 내보내기** — 구간(IN/OUT)·모드·코덱·CRF·미니맵·경기 시계.
16. 분석 메뉴 **선수 히트맵/활동량 리포트** — `<이름>_report/` 에
    팀/선수 히트맵 PNG + players.md (병합 품질에 의존 — 3번 후에).
    CLI: `python scripts/report.py <pano.mp4>`.

## 7. 파인튜닝 (선택, Windows — devlog 033)

```
python scripts/make_ball_dataset.py <pano.mp4>          # 검수 라벨 → 타일
python scripts/finetune_ball.py <...>/dataset.yaml       # yolo11m 짧은 학습
python scripts/ab_source_detect.py <proj.json> <pano.mp4> --weights <best.pt>
```
원경 recall 개선 확인 시 등록 방식 결정 (공 1클래스 — 선수 검출 대체 불가).
