- 날짜: 2026-07-19

- 관련: [018](20260719_018_click_modes_radar_export.md)

## 타임라인 내보내기 구간 (IN/OUT)

- 타임라인 **우클릭 메뉴**: "내보내기 시작/끝 지점", 구간 해제, 여기로
  이동. **I / O 단축키** = 현재 위치를 시작/끝으로 (NLE 관례).
- 표시: 구간 밖은 어둡게, IN(초록)/OUT(주황) 세로선 + 라벨.
- 역전 입력은 정규화, `.ptz.json` "export_range" 저장.

## 내보내기 대화창 (ExportDialog)

하단 바의 코덱/CRF/레이더 컨트롤을 대화창으로 이동 (바에는 모드·원경
줌·내보내기 버튼만). 대화창 구성:

- **구간**: 마커 구간(시각·길이·프레임 수 표시, 기본 선택) / 전체.
- 모드(미리보기 콤보와 동기화), 코덱, CRF/CQ, **미니맵 체크박스**
  (우하단 반투명 탑다운, QSettings 기억), 출력 파일(기본 이름에 구간
  시작초 포함) + 찾아보기.
- **코덱 기본값**: GPU 인식 시(목록에 있으면) **hevc_nvenc**, 직전
  선택이 있으면 그것 우선 (QSettings). CRF 도 기억.

## 구간 렌더 (core)

render_plan 에 start/end — cap 을 start 로 시크해 [start, end) 만
인코딩, **오디오도 -ss 로 같은 오프셋** (-shortest 로 끝 정렬).
진행률 분모는 구간 길이.

## 편집 화면 레이더 비율

RadarView 위젯 폭을 표시 범위의 실제 비율로 고정 — 캘리브레이션 후엔
경기장(예: 100×62m)+여백 비율 그대로, 이전엔 기본 범위(110×75m) 비율.
내보내기 미니맵과 동일한 등방 축척.

## 상태

- 테스트 55개. 참고: CPU libx264 도 체감 빠름 — 병목이 인코딩이 아니라
  6K 파노라마 디코딩/크롭이라 GPU 이득은 주로 HEVC 화질/용량.
