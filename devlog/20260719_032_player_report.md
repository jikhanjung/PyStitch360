# 032: 선수 히트맵/활동량 리포트 (P03-4)

- 날짜: 2026-07-19
- 종류: 구현
- 관련: [P03](20260719_P03_highlights_roadmap_plan.md) 4번 항목,
  [030](20260719_030_tracklet_merge.md) (병합 대표 tid 기준 — 순서 유지 이유)

## core/report.py (신규)

- `player_field_tracks` — 병합 대표 tid 로 합친 필드 궤적.
- `movement_stats` — **0.5s 창 평균 위치로 리샘플** 후 거리/속도 계산.
  검출 지터(±0.3m)가 랜덤워크로 거리를 부풀리는 것을 막는 핵심
  (raw 합산 대비 합성 테스트에서 부풀림 <1% vs 수십%). 11 m/s 초과
  스텝은 트래커 점프로 거리에서 제외, 2s 넘는 공백은 잇지 않음.
  반환: 관측 시간, 이동거리, 평균(활동 시간 기준)/최고 속도.
- `heatmap_grid` + `render_heatmap` — 1m 셀 점유 → 가우시안 블러 →
  TURBO 컬러맵을 잔디 배경에 점유 비례 알파로 합성, `field_outline`
  재사용으로 경기장 선(외곽·중앙선·서클·페널티박스). **등방 px/m** —
  경기장이 입력 실측 비율 그대로. 타이틀은 cv2 제약으로 ASCII.
- `generate_report` — 팀1/팀2 히트맵(GK 포함) + 선수별(팀 역할,
  검출 ≥150, 상위 30명) 히트맵 PNG + players.md 요약표(한글: 팀명·
  병합 (+N)·관측/검출/거리/평균/최고).

## 진입점

- GUI: 분석 메뉴 "선수 히트맵/활동량 리포트" → `<파노라마>_report/`
  에 생성 (캘리브레이션 필수, 병합·역할은 현재 검수 상태 반영).
- CLI: `python scripts/report.py <pano.mp4> [--out DIR] [--min-det N]`
  — .ptz.json 의 역할·병합·랜드마크로 동일 결과.

## 검증

- 합성 테스트 (통과): 등속 300m ±2%, 지터 부풀림 억제, 점프 제외,
  정지=0m, 그리드 좌표계, 병합 궤적 합침, 파일/markdown/등방 비율,
  렌더 육안 확인 (선·블롭·타이틀).
- **실데이터 검증 (Windows, 남음)**: 병합 품질(030)에 의존 — 병합
  후 선수별 이동거리가 상식 범위(경기당 수 km)인지, 히트맵이 포지션과
  맞는지. P03 순서(3→4)를 유지한 이유.
