# 027: 하이라이트 자동 추출 + 일괄 클립 내보내기 (P03-1)

- 날짜: 2026-07-19
- 종류: 구현
- 관련: [P03](20260719_P03_highlights_roadmap_plan.md) 1번 항목

## 구현

### core/highlights.py (신규)

이벤트 소스 → 규칙 구간 → 병합·점수화:

- `build_highlights(duration, kickoffs, whistles, signals, air_events,
  speed_events, user_events)` — P03 표의 규칙 구간(킥오프 [−10,+20],
  기 신호 [−8,+7], 롱 휘슬 [−8,+5], 공중볼→박스 [발사−4,착지+6],
  속도 급증 [−3,+5], 사용자 [−8,+8])을 펼치고 간격 <3s 를 병합.
  점수 = 가중 합 (user 5.0 > kickoff 3.0 > foul/offside 2.5 >
  whistle/air 2.0 > speed 1.5), 라벨은 최고 가중 규칙(사용자 라벨 우선).
  킥오프 ±5s 휘슬은 휘슬 규칙에서 제외 (중복 방지).
- `ball_speed_events(t, gxy)` — 수락 공 필드 궤적에서 20 m/s 이상
  2스텝 지속 + **순변위 게이트**(구간 순변위 속도 ≥ 0.6×문턱).
  순변위 게이트가 없으면 한 샘플 왕복 지터가 연속 2스텝 초과를 만들어
  가짜 이벤트가 됨 (합성 테스트에서 확인). >45 m/s 는 검출 점프로 기각.
- `airborne_box_events(segments, L, W)` — .events.json "airborne" 의
  탄도 피팅에서 착지점(p0+vT)이 경기장 끝 25 m 안이면 슛/크로스 후보.
- `carry_states(new, old)` — 재생성 시 겹침(짧은 쪽 50%)으로 기존
  수락/제외 상태 승계, 수락 항목은 사용자 조정 경계 유지, 비겹침
  수락/제외 항목 보존 → **비파괴 재생성**.

저장: `.events.json "highlights"` = [{t0, t1, kinds, label, score, state}]
(state: cand|accept|reject, 초 단위).

### GUI

- 분석 메뉴 "하이라이트 후보 생성 (이벤트 융합)" → `detect_highlights()`:
  .events.json (kickoffs/linesman_signals/airborne) + 호각 트랙 +
  수락 공 궤적 + 사용자 이벤트 융합. 캘리브레이션 없으면 공중볼/속도
  규칙만 생략하고 진행.
- 타임라인 이벤트 레인에 구간 바: 후보=호박색, 수락=초록, 제외=회색
  (라벨 표기, 이벤트 마커는 위에 그대로). 클릭=구간 시작으로 이동,
  우클릭 = 수락/제외/후보 되돌림/삭제 + **경계 조정은 IN/OUT 마커 재사용**
  ("IN/OUT ← 하이라이트"로 불러 조정 후 "하이라이트 ← IN/OUT"으로 커밋).
- 분석 메뉴 "하이라이트 일괄 내보내기..." → `HighlightExportDialog`
  (구간 체크 목록 — 수락 항목 기본 체크, 수락이 없으면 전부 체크 —
  출력 폴더, 코덱/CRF/미니맵은 기존 ExportDialog 규칙 재사용) →
  `HighlightBatchWorker`: build_plan 1회 계산 후 구간별
  `<이름>_h01_킥오프.mp4` 순차 render_plan (start/end — 오디오 싱크
  포함), 진행률 = 전체 구간 프레임 합, 기존 취소 버튼으로 취소.

  게이트. 스크립트: scratchpad/test_highlights.py (일회성).
- **실데이터 검증 (Windows, 남음)**: 후보 ~15±10개 생성, 과반 유의미,
  3개 구간 일괄 내보내기 (오디오 싱크 확인) — P03 완료 기준.

## 추가: 전용 하이라이트 레인 + 수동 추가 (같은 날)

- 하이라이트 구간 바를 이벤트 레인에서 분리해 **"하이라이트" 전용
  레인**(9번째)으로 — 이벤트 마커와 안 겹쳐 검수·수작업이 쉬움.
  구버전 저장 레인 높이(8개)는 기본 높이로 패딩해 사용자 조정 보존.
- 우클릭 "여기에 하이라이트 추가 (±8초)" / "IN/OUT 마커 구간 → 새
  하이라이트" — 자동이 못 잡는 장면(빗나간 슛 등) 수동 등록. 수동
  추가는 바로 수락 상태라 재생성해도 carry_states 로 보존된다.
