# 026: UI 응답성 — 프리즈 수정, busy 커서/로그, 뜬 공 레인, 캐시

- 날짜: 2026-07-19
- 종류: 작업 기록
- 관련: [025](20260719_025_airborne_3d.md)

## PTZ 탭 프리즈 수정 (실사용 보고)

공중볼 스캔이 UI 스레드에서 전체 궤적을 돌며 탭 열기 ~1분 "응답 없음":

- **스캔 최적화**: 창 길이 전수 축소 → 4개 후보만, 싼 직선 잔차 사전
  게이트(<1m 굴림은 피팅 생략). 실측 60s → **5s** (구간 36→20,
  남은 것이 고신뢰). 합성 테스트 8.6s → 0.6s.
- **AirborneWorker**: 남은 비용도 백그라운드 — UI 차단 ~1ms. 세대
  카운터로 낡은 결과 폐기.
- **파일 캐시** (사용자 요청): 결과를 `.events.json` "airborne" 에
  저장 (무효화 키 = 샘플 수·후보 수·무시·승격 해시). 다음 열기엔
  스캔 생략(구간 재투영만) — "캐시 재사용" 로그.

## busy 커서 + 작업 로그 (사용자 요청)

`_busy(msg)` 컨텍스트: 시작 로그 "[작업] …" + 웨이트 커서 + 0.5s
이상이면 완료 로그(소요 시간). 적용: 파노라마 열기(호각 읽기·목록
갱신), 사이드카/분석 JSON 읽기(수 MB), 분석 저장, 킥오프 검출, 주변
재검출, 흰 선 검출. 백그라운드 워커들은 기존 시작/진행 로그 유지.

## 타임라인 "뜬 공" 레인 (사용자 아이디어)

공 레인 아래 8번째… 아니 3번째 레인: 공중 구간을 하늘색 바로, 폭이
되면 정점 높이 라벨(예: "2.1m"). AirborneWorker 완료 시 갱신.
레인 인덱스 전면 시프트 (팀 3/4, 기타 5, 호각 6, 이벤트 7).

## 레이더 공 크기 (사용자 요청)

편집 레이더 3px → **5px 반경 + 어두운 외곽선**, 내보내기 미니맵도
r+1 확대 + 외곽선 — 잔디 위 대비 개선.
