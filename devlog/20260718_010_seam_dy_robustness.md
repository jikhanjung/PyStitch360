# 010: 심 세로 어긋남 보정, pitch/yaw 의존 오동작 수리

- 날짜: 2026-07-18
- 종류: 작업 기록
- 관련: [009](20260718_009_gui_polish.md)

## 증상 (사용자)

refine_seam 의 세로 어긋남 보정이 어떤 pitch/yaw 에서는 잘 맞고 어떤
경우에는 안 맞음.

## 진단 (600s 프레임, scale 0.5, 구성 6종)

측정값 자체가 대부분 가짜였다:

- dy 측정치가 **±28px(= 탐색 margin)에 포화** — 심 근처의 방사형 잔디
  줄무늬는 심에서 거의 수직선이라 세로 매칭이 원리적으로 불가한데
  (aperture problem) matchTemplate 점수는 높게 나옴.
- 유효 타일 5~11개 위에 자유 2차 다항식을 피팅해 측정 없는 최하단으로
  외삽 → 구성에 따라 -44 ~ +34px 의 임의 워프. dy rms 10~23px (비물리적).

## 수정 (`render.py`)

1. **aperture 기각**: 타일의 가로 에지 에너지(|Sobel_y| 평균 < 6) 부족 시 기각.
2. **포화 기각**: |dy| ≥ margin−2 인 매칭은 가짜로 보고 버림.
3. **물리 모델 피팅**: 시차 ∝ 1/거리 이므로 수평선 아래에서
   dy = a + b·tan(−el), 위에서는 상수 a (elevation 은 Renderer 에 저장한
   el0/el1 에서 행별 환산). pitch/세로범위가 바뀌어도 elevation 공간에서
   일관되고 외삽도 물리적으로 타당. 최소 타일 3→4개.

## 결과 (같은 6종 구성)

- dy 측정 범위 ±28(포화) → **±4px** (물리적), rms 1.5~2.5px → 보정 후 0.0~0.5px.
- pitch+8: 유효 타일 2개 → 보정 생략 (이전: 쓰레기 워프 주입, 이제: 안전 무동작).
- yaw+6: 실제 시차 증가로 dy −20px 관측, rms 9.5→4.6px — **yaw 를 틀면 심이
  대칭축을 벗어나 진짜 시차가 커진다**는 앞선 분석의 실증. yaw 조정은
  몇 도 이내 권장.
- 심 밴드 육안: 6종 전부 하프라인 연속.
