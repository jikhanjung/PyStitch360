# 041: 멀티캠 동기화 코어 (P06-1)

- 날짜: 2026-07-20
- 종류: 구현
- 관련: [P06](20260720_P06_multicam_fusion_plan.md) 1단계

## core/sync_multi.py (신규)

시계 모델 t_A = offset + drift·t_B.

- `sync_by_whistles` — 호각 온셋 전체 쌍 차이의 히스토그램 투표로
  지배적 오프셋 → tol 내 1:1 그리디 매칭 → IRLS(Huber 0.2s) 선형
  피팅. 오검출(한쪽에만 있는 이벤트)과 음속 지터(±0.3s)에 강건.
  드리프트는 매칭 쌍의 시간 분포에서 나온다.
- `refine_sync_by_ball` — 공 궤적(필드 좌표)을 시계 모델로 겹치고
  추가 오프셋 δ 를 조밀 스캔(0.05s → 0.01s), 거리 **중앙값** 최소화.
  공간 정합 미정이면 직사각형 대칭 4가지를 함께 시험해 판별 —
  동기화와 공간 정합(P06-2)의 닭-달걀을 한 번에 푼다. 보간은 갭
  브리지 방지(근처 0.5s 내 실측 없으면 무효).
- `refine_clock_by_ball` — B 시간축을 창 3개로 나눠 창별 δ 를 재고
  δ(τ) 선형 피팅 → 오프셋+드리프트 동시 정밀화.

## scripts/sync_cams.py (실데이터 검증, Windows)

두 영상의 .whistle.json 으로 거친 동기화, 양쪽에 분석+캘리브레이션이
있으면 공 궤적 정밀화까지 → A 영상 .events.json "sync" 저장.
AX700 은 캘리브레이션 전이므로 당분간 호각 단계까지 (P06-3 후 정밀).

## 검증 (tests/test_sync_multi.py — 통과)

합성: 오프셋 127.43s + 드리프트 40ppm, 호각 18개(음속 지터 ±0.3s,
오검출 6개 혼입), 공 10Hz(노이즈 0.3m, 25% 결손):

- 거친 단계: 오프셋 <0.25s, 드리프트 <5e-4 — 설계대로 프레임 정밀은
  못 하고,
- 공 정밀화 후: **경기 전 구간(t=0/900/1650s)에서 오차 < 1프레임
  (33ms)** — P06-1 완료 기준 충족.
- 대칭 4종(항등/X반전/Y반전/180°) 전부 정판별, 겹침 없음/이벤트 부족
  등 실패 경로 None 반환.

## 남은 것 (Windows)

실경기 파노라마↔AX700: whistle.py 를 AX700 오디오에 실행(마이크
특성 확인) → sync_cams.py 거친 동기화 잔차 확인. 공 정밀화는 P06-3
(AX700 필드 정합) 이후.
