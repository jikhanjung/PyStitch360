# 011: PTZ 탭 구현 — 트랙바, 트랙 무시, 크롭 미리보기, 와이드 모드

## 4번 탭 "가상 PTZ" (사용자 피드백 반영 이력 포함)

- 파노라마 열기(최근 폴더 시작) → 분석 캐시/키프레임 사이드카 자동 로드.
  프로젝트 파일은 열린 파노라마 경로만 기억(사이드카는 파노라마 종속).
- **트랙 단위 오검출 처리**: `accept_ball_tracks()` — 후보를 최근 5점
  평균 기준(연결 상한 850px)으로 트랙화, 정지미끼/지속고립/시간중복 기각.
  3분 클립 낙엽 응시 2548→53프레임. 완주 분석(23,689샘플): 원시 검출
  58% → 트랙 242→189 수락, 확정 궤적 19% (나머지는 줌아웃+키프레임 몫).
- **트랙바**: 슬라이더 위에 수락 트랙(초록)/무시(빨강)/키프레임(주황)
  표시, 클릭 이동. "공이 어디서 검출됐는지" 시각화 요청 반영.
- **"현재 공 트랙 무시" 버튼**: 오인식 트랙을 통째로 기각 (프레임 단위
  수동 삭제 불가 지적 반영). 사이드카 v2 {keyframes, ignores}, 구버전
  리스트 형식 호환.
- **크롭 창 미리보기**: 키프레임/무시/모드 변경 시 계획을 백그라운드
  재계산(600ms 디바운스)해 현재 프레임에 render_plan 과 동일 클램프의
  크롭 사각형 오버레이 — "결과가 어떻게 보일지" 확인 요청 반영.
- **와이드 감상 모드** (2560×1080): 원근 보정판 소스용 — 폭 최대 고정,
  σ=3s 완만 팬. PTZ 16:9 는 원본 소스(화질), 와이드는 보정판(원근 편안).

## NVENC 발견 (WSL)

- 과거 devlog 003 "WSL 에 NVENC 없음" → **현재는 동작** (드라이버/ffmpeg
  업데이트). hevc_nvenc 는 5904px 파노라마 가능 (libx264 대비 5배),
  h264_nvenc 는 4096px 폭 한계 → PTZ/와이드(≤2560px)용.
- Windows 에서 콤보에 libx264 만 나온 원인 = ffmpeg 미발견 폴백 (winget
  설치 후 PATH 미갱신). `ffmpeg_bin()` 으로 탐색 강화 + 전 호출부 적용.

## 기타

- wait cursor: setOverrideCursor 직후 processEvents 로 즉시 반영
  (블로킹 직전 커서가 안 바뀌는 문제).
- 프레임 크기 홀수 문제, 컨테이너 메타데이터 프레임 수 오차(±1%) 허용.
