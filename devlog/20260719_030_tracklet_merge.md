- 종류: 구현

## 설계 원칙: 되돌리기 우선 (사용자 요구)

팀 분류 오류가 종종 있으므로 "먼저 다 고치고 병합"이 아니라 **병합
먼저, 수정은 나중에**가 기본 흐름:

- 병합 맵 {tid: 대표}만 .ptz.json "merges" 에 저장 — 분석·역할 데이터
  불변 (비파괴). 그룹 해체/멤버 분리 시 원래 분류로 자동 복귀.
- 유효 역할은 **대표 기준** (`_role_of` 가 대표 role 우선 해석) —
  팀이 잘못 분류된 조각도 그룹에 넣으면 표시가 바로잡힌다.
- 중간에 빠뜨린 조각은 나중에 그룹 멤버 + 조각을 함께 선택해 병합
  (union 방식이라 기존 그룹에 합류).

## core/tracklets.py (신규)

- `tracklet_summaries(analysis, calib)` — tid별 {t0, t1, 첫/끝 필드
  좌표(edge 5개 중앙값), s-가중 색 벡터(classify_teams 특징 공간), n}.
- `suggest_links(summ, roles)` — P03 조건: (a) 시간 겹침 없음(0.3s
  지터 허용), (b) Δt<3s & 필드 거리 < Δt×8m/s+2m, (c) 색 거리<60,
  (d) 유효 역할 일치. 비용(거리·색 정규화 합) 순 그리디, 트랙릿당
  후속/선행 각 1개 — 체인.
- `merge_map(links, n_det)` — union-find 연결 성분, 대표 = 검출 최다.
  기존 병합 링크를 함께 넣어 재실행·수동 병합·조각 추가가 전부 같은
  경로.

## GUI

- 분석 메뉴 "트랙릿 병합 제안 (ReID 라이트)" — 기존 병합(수동 포함)
  유지하며 새 링크 union. 캘리브레이션 필요 (필드 좌표).
- 선수 목록: 대표 행 "#tid (+N)" + 합산 구간/검출 수, 멤버는 "↳" 들여
  쓰기로 계속 표시 (선택 가능해야 분리·조각 추가가 됨).
- 목록 우클릭: "선택 N개 병합" / "#tid 그룹에서 분리 (원래 분류로)" /
  "그룹 해체". 역할 지정은 멤버를 선택해도 대표에 적용 (그룹 = 한 사람).
- 역할 초기화(reset_edits "roles")에 병합도 포함.

## 검증

- 합성 테스트 (통과): 체인 제안, 색/거리/시간/역할 게이트, 후속 1개
  제한, 대표 선정, 기존 병합에 조각 추가, 요약 생성(직접 구성 calib).
- **실데이터 검증 (Windows, 남음)**: 2,135 트랙릿 → 1/5 이하 목표,
  선심 커버 공백 감소, 오병합 육안 표본 드묾 (P03 완료 기준). 문턱
  (색 60, 속도 8m/s, 갭 3s)은 실데이터 보고 조정.
