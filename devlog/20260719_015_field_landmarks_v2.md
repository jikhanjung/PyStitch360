## 랜드마크 v2

- **드래그 이동**: 찍힌 마커를 잡아 끌면 이동, 이동 중 라이브 재피팅
  (외곽선·레이더 즉시 갱신). 마커 위 클릭(드래그 미수)은 무동작.
- **사이드라인 선 제약**: 카메라 위치의 중앙선×사이드라인 교차점이 안
  보일 때, 크게 확대돼 보이는 **가까운 사이드라인 위 아무 점 2개**
  (sideline_near_l/r)로 그 선(Y=-폭/2)을 고정. 위치를 모르는 점이라
  점당 방정식 1개(해당 열의 광선이 선과 만나는 예측 행 vs 클릭 행).
  최소 조건: 위치 점 3+ 이고 총 방정식 8+ (위치 4개 또는 3개+선 2점).
- **페널티박스 추가**: 골라인 위 모서리 4 (pen_[lr]_[far|near]) +
  박스 안 모서리 4 (16.5m, ±20.16m). field_outline 에 박스 3변 포함 —
  오버레이/피팅 확인용.
- **권장 순서 = 목록 순서**: 1단계 최외곽 선 위 ~10점(코너4 → 중앙선×
  사이드라인2 → 사이드라인 보조2 → 골라인 페널티 모서리4) → 2단계
  내부(센터서클2, 박스 안4). 목록에 번호 표시. 외곽 6~7점이면
  외곽선이 잡힌다 (점 추가마다 증분 재피팅).

## 매칭 휴리스틱 재설계 (오배정 수정)

랜드마크가 18종으로 조밀해지자 기본 카메라 모델의 거리 오차(원경
수십 m)가 이웃 간격(코너↔페널티 13.8m)을 넘어 오배정 발생 — 2단계로:

- **피팅 전**: 외곽 8종만 후보. 카메라 기준 **방향(요)은 강하게**
  (σ≈5.7°), **거리는 로그 비율로 느슨하게**(σ≈0.4) 비교 — 높이 오차는
  모든 점을 같은 배율로 밀므로 방향이 훨씬 신뢰할 수 있다.
- **피팅 후**: 전체 랜드마크를 화면에 투영해 **픽셀 최근접**.
- 검증(합성, 기본 가정과 다른 카메라 h5.0/5m 치우침/회전): 외곽 7점
  7/7 → rms 0.02px → 내부 10점 10/10, 최종 17점 rms 0.03px.

## 찍기 모드 UX

- **오브젝트 숨김**: 랜드마크 찍기 모드에서는 크롭 박스·공 후보·
  키프레임·선수 박스를 모두 숨김 — 경기장 선·마커만 보여 클릭 대상이
  헷갈리지 않는다. 박스 편집도 비활성.
- **전체 삭제** 버튼 (확인 후), 선택 삭제와 별도.
- **다음 점 안내**: 찍기 모드에서 상태 라벨에 "다음 찍을 점 → n. 이름
  [태그]" 표시 + 목록에서 해당 행 자동 하이라이트. 점을 찍을 때마다
  다음 미지정 랜드마크로 진행.

## 주변 사람 재검출 (우클릭)

빈 곳 우클릭 → "이 주변 사람 재검출": 캘리브레이션으로 그 자리 사람
(1.8m)의 픽셀 키를 예측해 **키의 ~6배 정사각 크롭**(마진 넉넉히,
320~1800px)을 네이티브 해상도로 잘라 YOLO(person, conf 0.1)를 돌린다.
프록시 표시 중이면 원본 영상에서 프레임을 읽음(캡처 캐시). 검출된
사람은 수동 검출(extra, ID 900001+)로 추가되고 즉시 역할 지정 메뉴
(팀1/팀2/GK/심판)가 뜬다. 기존 박스와 겹치면 중복 추가 안 함.
수동 검출은 선수 목록·레이더·박스 오버레이에 통합되고 `.ptz.json`에
저장, 우클릭으로 개별 삭제. 분석 메뉴의 역할 초기화/전체 초기화에
포함.
