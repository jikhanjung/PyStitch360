## 심판 위치 자동 분류 (classify_referees)

비팀 트랙릿의 필드 좌표 통계로 역할 제안 (사용자 아이디어: 주심은
필드 안 상주, 선심은 각자 사이드라인):

- **선심**: 사이드라인 ±3.5m 상주 + Y 요동(IQR) ≤3m + 터치라인 방향
  이동 ≥8m. 근측/원측 분리. **주심**: 필드 내부 비율 ≥75% + 활동 범위
  X≥15m·Y≥6m. 트랙릿 조각별 판정 — ID 갈라져도 조각마다 잡힘.
- 실측: 주심 18 / 근측 선심 11 / 원측 선심 2 트랙릿. 사용자가 수동
  지정한 심판 트랙릿(373)이 자동 분류에 포함 — 정합 확인.
  `.events.json` "referees" 저장.

## 선심 기 신호 (포즈 추정)

호각(≥20dB)마다 [t−10s, t+6s] 창에서 근측 선심 크롭(네이티브, 키
×2.2)을 **YOLO11n-pose**(COCO 17키포인트)로 추정:

- `arm_pose_scores`: (올림, 뻗음) = (어깨−손목 세로차, |손목−어깨
  가로거리|) / 몸통. `classify_flag_signal` 시퀀스 분류 —
  **foul** = 올림 ≥0.75 지속 ≥0.8s (들고 흔듦 포함),
  **offside** = 올림 구간 *이후* 수평 지시(올림≈0, 뻗음≥0.6) ≥0.8s.
  순서 조건이라 팔 벌린 자세 오탐 없음. 원측 선심은 해상도 한계로
  --far 옵션일 때만.
- 실측 (23호각, 1.8분): 근측 커버 17개. **13:59.8 호각에서 기 신호
  검출** (올림 1.03, 1.41s) — 크롭 확인 결과 14:04~06 기 수직 유지,
  실제와 일치. 나머지는 팔 내림(주심 단독 판정) — 자연스러움.
  이 경기엔 오프사이드 수평 지시 사례 없음 — 다음 경기에서 검증.
- 커버 0인 호각 6개: 카메라 설치 구간(2), 종료 후(2), 선심 트랙릿
  조각 공백(2) — 조각 커버리지가 한계, ReID/시드 전파가 도움될 것.
- `.events.json` "linesman_signals" 저장 (팔 시계열 포함 — 문턱 변경
  시 재추정 불필요).
