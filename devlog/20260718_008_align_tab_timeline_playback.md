# 008: 정합 탭 타임라인 + 파노라마 동영상 재생

- 날짜: 2026-07-18
- 종류: 작업 기록
- 관련: [007](20260718_007_ptz_v2_two_pass.md)

## 사용자 피드백

1. 정합은 현재 프레임 기준인데 프레임을 옮기려면 1번(동기화) 탭에 가야 함
   → 정합 탭에도 타임라인 필요.
2. 정합이 끝나면 정지 프레임이 아니라 **영상으로** 결과를 보여줄 것.

## 구현

- **정합 탭 타임라인**: 1번 탭 슬라이더와 양방향 동기(QSlider 상호 연결,
  같은 값 setValue 는 재발신 안 하므로 루프 없음), ±1/±10 스텝, 시간 표시.
  프레임 이동 시 정합 탭이 활성이면 파노라마 미리보기도 자동 갱신.
- **PlaybackWorker** (`gui/workers.py`): 렌더러를 시작 시 1회만 구성
  (게인·refine_seam 포함), **자체 ChapteredVideo 인스턴스**로 순차 디코딩
  (GUI 스레드 디코더와 충돌 없음). display_fps(기본 10)에 맞춰 프레임을
  건너뛰고, 건너뛰는 프레임은 새로 추가한 `ChapteredVideo.grab()`(색변환
  없는 디코딩, 챕터 경계 자동 통과)으로 처리. 실시간 페이스 유지, 렌더가
  느리면 자연 감속.
- **자동 재생**: 정합 완료(_align_done) 시 바로 재생 시작. ▶/⏸ 토글,
  슬라이더 드래그·스텝·탭 전환·정합/내보내기 시작 시 자동 정지.
- 재생 중 슬라이더는 재생 위치를 따라가고, 정지 시 그 위치에서 cur_imgs
  복원 → 그 자리에서 바로 재정합 가능 (충격 이벤트 워크플로우와 맞물림).

## 성능 (WSL, 보정판 렌더와 CPU 경합 중 측정)

표시 2.5fps / 0.23x 속도. 병목은 4K×2 디코딩(경합 중 68ms/프레임).
경합 없으면 0.5~0.7x 예상. 더 올리려면 저해상도 프록시 디코딩(LRV 파일
활용!) 검토 — GoPro 가 챕터마다 .LRV(848×480 저해상도 동반 파일)를 남기니
재생 전용으로 쓰면 실시간 초과도 가능. 다음 후보로 기록.

## E2E + 프로젝트 저장

실영상 E2E: 오디오 오프셋 +0.068s(conf 34.8) → 정합(인라이어 816/1106,
잔차 0.14°) → 12프레임 재생 확인. 이 결과를 **`projects/match_20260712.json`
으로 저장** — 어제 완주 정합이 프로젝트로 남지 않았던 공백 해소. 이제 GUI
에서 열면 세그먼트·오프셋·원근 설정(k=0.3, m=1.3)이 그대로 복원된다.
