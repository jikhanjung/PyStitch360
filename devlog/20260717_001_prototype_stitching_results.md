## 자동 보정 3종 (+ 사용자 오프셋 `--pitch/--roll/--yaw`)

1. **상대 회전**: 겹침 영역 SIFT → 광선 변환 → RANSAC+Kabsch.
   인라이어 ~800개, 잔차 중앙값 0.13°.
2. **수평(pitch/roll)**: 먼 쪽 터치라인 검출(흰선 + 위는 수풀/트랙 + 아래는 잔디)
   → elevation(yaw) 을 `a + b·cosφ + c·sinφ` 로 피팅 → b,c ≈ 0 이 되도록 보정.
   보정 부호를 관례로 가정하지 않고 **시험 회전 ±2° 로 수치 야코비안을 측정해
   2x2 선형계를 푸는 방식** (부호 실수 원천 차단). 2회 반복으로 수렴:
   pitch +29.1°, roll -5.9°.
   - 교훈: 해석적 부호 유도로 `pitch += b; roll += c` 방식은 roll 부호가 반대라
     발산했음 (-11.7° → -29.6°). 야코비안 방식으로 교체 후 안정.
3. **센터링(yaw)**: 하프라인(설치점에서 방사 방향 → 원통 투영에서 수직선) 검출.
   저해상도 파노라마에서 열별 흰색 픽셀 합의 피크. 이번 세션 -4.5°.

## 영상 처리 성능 (10초 = 299프레임, 5906×1680 출력)

| 단계 | 프레임당 | 비고 |
|---|---|---|
| 디코딩 (4K×2) | 19ms | cv2.VideoCapture |
| 워핑 (remap×2) | 36ms | CV_16SC2 고정소수점 맵 |
| 게인+블렌딩 | 51ms | cv2.multiply + cv2.blendLinear |
| 인코더 대기 | 40ms | libx264 fast CRF19, ffmpeg 파이프 |
| **합계** | **146ms → 6.5fps** | 실시간의 22%, 90분 경기 ≈ 7시간 |

- numpy float 브로드캐스트 블렌딩(172ms)을 cv2 네이티브로 교체해 3.7→6.5fps.
- 추가 여지: 스레드 파이프라이닝(읽기/워핑/인코딩 중첩)으로 ~2배,
  NVENC 하드웨어 인코딩, cv2.cuda remap. GUI 단계에서 적용 검토.
