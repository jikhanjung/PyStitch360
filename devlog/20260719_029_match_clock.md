## 경기 정보 (match_info, .ptz.json)

분석 메뉴 "경기 정보 (시계 앵커·중단 구간)..." 대화창:

- **하프**: 전반/후반 (현재 검증 영상은 후반만)
- **하프 길이**: 분 단위 (이 경기는 30분 — 45분 고정 아님)
- **킥오프 앵커**: .events.json 킥오프 목록(신뢰도 표기) / 현재 프레임 /
  미지정. 저장된 앵커가 목록에 없으면 보존 항목으로 표시.
- **누적 표시**: 후반 시계에 하프 길이를 더해 30:00(또는 45:00)부터.
- **중단 구간**: 타임라인 우클릭 "IN/OUT → 경기 중단 구간 (시계 정지)"
  으로 추가, 구간 안에서 우클릭으로 삭제. 회색 세로 밴드로 표시.
  hydration break(이 경기: 킥오프 +15분, 중단 동안 시계 정지)용.

부수 수정: `_load_sidecar` 가 user_events 를 리셋하지 않아 다른 영상의
이벤트가 남던 문제 수정 (match_info 도 함께 리셋).

## 시계 렌더 (core/ptz.py)

- `clock_string(clock, frame)` — 앵커 이전은 시작값 고정, 중단 구간에서
  정지. **표기는 분:초 누적** (시:분 아님): 후반 30분 하프면 30:00→60:00,
  정식 경기 연장이면 분이 90, 120까지 커진다 (예: "2H 52:10",
  "ET 120:30").
- 스코어: 사용자 이벤트 라벨 "골1"/"골2" 를 팀1/팀2 득점으로 집계 —
  득점 프레임부터 "HOME 1-0 AWAY" 로 바뀐다. cv2 폰트는 한글 불가라
  태그는 1H/2H, 비ASCII 팀 이름은 T1/T2 로 대체. 골 이벤트 없으면
  시계만.
- `_draw_clock` — 좌상단 반투명 검정 박스 + 흰 글자, 미니맵과 같은
  스케일 규칙 (out_h 비례 폰트, out_w//96 마진).
- `render_plan(clock=...)` — PTZ 단일 내보내기(ExportDialog 체크박스,
  설정 기억)와 하이라이트 일괄 내보내기 양쪽에서 사용. 앵커 미지정이면
  체크박스 비활성 + 안내 툴팁.

## 검증

- 합성 테스트 (통과): 사용자 시나리오 그대로 — 후반 30분 하프, 킥오프
  +15분에 2분 hydration break → 브레이크 중 45:00 정지, 재개 후 이어감,
  종료 60:00. 연장 120:30 분 누적, 스코어 시점별 반영, 1080p/720p 렌더
  스모크.
- **실데이터 검증 (Windows, 남음)**: 실제 후반 영상에서 경기 정보 입력
  → 브레이크 구간 지정 → 내보내기 후 시계·스코어 확인.
