## GK 단일성 (classify_teams)

사용자 관찰: "GK1/GK2 는 각각 하나 — GK1 이 골대 근처에 있으면 다른
위치에 또 GK1 이 있을 가능성이 낮다." 색 전파가 같은 GK 역할을
**시간상 겹치는** 여러 트랙릿에 붙이면 오분류라는 뜻:

- 전파 후 필터: GK(3/4)별로 시드 우선 → 역할 색 근접 순으로 남기고,
  이미 남은 것과 0.5s 넘게 겹치는 전파분은 기본 군집으로 되돌림.
- 시간 비겹침 조각(ID 갈라짐)은 전파 유지 — 전파의 원래 목적.
- 심판(5)은 주심+선심 3명이라 제외. 위치 사전지식(골문 근접)은
  calib 의존이라 이번엔 미적용 — 겹침 오탐이 실데이터에서 남으면 추가.
- 기존 테스트(test_ptz_plan)의 "동시 존재 GK 조각" 모델링은 물리적으로
  불가능한 시나리오라 순차 갈라짐으로 수정 (새 규칙이 걸러야 할 케이스).

## 휴리스틱 카탈로그 (docs/heuristics.md)

사용자 요청: "이런 약한 규칙들을 모아놓을 곳". E2E 검출이 좋아지면
불필요해질 도메인 규칙들을 규칙→근거→구현 위치→파라미터 표로 정리
(공/선수/병합/이벤트 4개 절). CLAUDE.md 에 등록 의무 지침 추가.

## 검수 UI

- **Space = 재생/정지** — 포커스가 아니라 **커서 위치** 기준 (타임라인·
  슬라이더·영상 패널 위에서만). 앱 이벤트 필터로 구현해 다른 위젯의
  Space 동작(목록 선택 등)은 통과.
- **선심 ARN/ARF 표기** — referee.py 의 근/원측 분류(.events.json
  "referees")를 GUI 가 읽어 오버레이 태그(REF→ARN/ARF)와 선수 목록
  ("선심 ARN")에 표시. 병합 그룹은 멤버 하나만 분류에 있어도 적용.
- **역할 메뉴 정리** — 우클릭 역할 지정에서 자주 쓰는 팀1/팀2/심판을
  위로, GK 를 구분선 아래로 (역할 버튼 행도 동일 순서).

## 타임라인 서브행 상한 제거

사용자 관찰: "팀 레인의 세 줄이 진짜 세 줄이 아니라 더 많이 겹쳐 있는
느낌" — 맞음. 구버전은 서브행 3개 상한이라 4번째부터 기존 행 위에
덮여 숨었다 (팀당 동시 트랙릿 ~11+). 상한을 없애고 동시 트랙릿
수만큼 서브행을 만들어 레인 높이를 행 수로 나눠 배치 — 겹침 은폐가
없어지고, 레인 경계 드래그로 높이를 키우면 행이 두꺼워진다.
