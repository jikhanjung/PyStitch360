## 테스트 승격 (tests/)

오늘 스크래치패드에서 검증한 합성 테스트를 pytest 로 이식 — 세션이
끝나도 회귀 방지가 남도록:

- `test_highlights.py` — 규칙 구간·병합·점수·라벨 우선순위·상태 승계·
  속도 이벤트(지터/점프 내성)·공중볼 착지 게이트 (P03-1)
- `test_clock.py` — 실경기 시나리오(후반 30분 하프, +15분 2분 브레이크
  정지), 분 누적 120:30, 스코어 시점 반영, 렌더 스모크 (P03-2)
- `test_tracklets.py` — 병합 체인·색/거리/시간/역할 게이트·후속 1개·
  union 대표 선정·조각 추가 (P03-3)
- `test_teams.py` — **팀 수정 격리 회귀** (devlog 031 버그) + GK/심판
  전파 유지
- `test_report.py` — 등속 300m ±2%, 지터 부풀림 억제, 점프 제외,
  히트맵 좌표계·등방 비율, 병합 반영 엔드투엔드 (P03-4)
- `test_labels.py` / `test_dataset.py` — 라벨 4종 분류, 데이터셋
  엔드투엔드 (합성 mp4, `slow` 마커) (P03-5)
- `conftest.py` — 합성 calib(직접 구성 카메라 모델)·팀/공 분석 픽스처.
  `pytest.ini` — testpaths + slow 마커.

WSL 실행 결과: 95개 중 93 통과. 실패 2는 기존 test_core.py 의 환경
의존 항목 (PyQt6 미설치, /mnt/d 드라이브 매핑) — Windows 에선 통과
조건이며 이번 변경과 무관 (변경 전에도 동일 실패 확인).

## 워크플로우 문서 (docs/ptz_workflow.md)

분석 메뉴가 8항목으로 늘어 순서 의존성 정리: 사이드카 파일 표 →
분석·검수 → 오디오/킥오프/심판 → 병합 → 경기 정보/득점 →
하이라이트 → 내보내기/리포트 → 파인튜닝(Windows). README 에 링크.
