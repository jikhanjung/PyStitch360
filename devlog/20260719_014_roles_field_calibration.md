## 선수 역할 분류 — 팀1/팀2/GK×2/심판

경기장 위 인원 구성(팀1, 팀2, 각 팀 GK, 주심·선심·4심)에서 GK 는 팀과
다른 색 옷을 입으므로 색 k-means 상위 2군집만으로는 못 가른다.
**사람은 일단 다 검출**하고(기존 conf 0.2 그대로), 역할은 사용자
시드 지정 + 색 전파로:

- `classify_teams(analysis, roles={tid: 역할})` — 시드 트랙릿들의 색
  중앙값을 역할 센터로 삼고, **어느 팀 색보다 역할 색이 가까우면 그
  역할**을 부여. ByteTrack ID 가 갈라져도 GK 한 명만 찍으면 같은 색의
  다른 트랙릿까지 따라온다. 시드 자신은 무조건 확정, 무시드는 기존
  0/1/2 동작 유지. 역할 번호 규약 ROLE_* (0/1 팀, 2 기타, 3/4 GK, 5 심판).
- 심판은 주심·선심·4심을 한 클래스(같은 색 계열)로 묶음.
- GUI: 색 6종(+GK 핑크/시안, 심판 노랑) 박스·레이더, GK1/GK2/REF 태그,
  선수 bbox **우클릭 → 역할 지정 메뉴**, `.ptz.json` 에 roles 저장.
- `tracklet_colors`: 트랙릿별 유니폼 대표색 (H 원형 경계 보정) —
  목록 스와치·팀 색 범례용.

## 하단 스트립 탭화 + 선수 탭

- 공 검수(두 목록 + ≫≪)를 "공" 탭으로, 새 "선수" 탭: 트랙릿 목록
  (유니폼 색 스와치 + 역할 + 구간 + 검출 수, 검출 수 순), 다중 선택 후
  역할 버튼 일괄 지정, 선택=해당 프레임 이동.
- 분석/역할 변경 시 **팀 색 범례**: 역할별 대표색 스와치 + 인원수를
  라벨과 로그로 표시 ("이 팀이 이 색").

## 경기장 캘리브레이션 (core/field.py)

레이더 절대화(로드맵)의 첫 단계. 랜드마크 8종: 코너 플래그 4,
중앙선×사이드라인 2, 중앙선×센터서클 2 — 일부 생략 가능, 최소 4점
(먼쪽 코너 2개 필수 권장).

- 모델: ground_positions 와 같은 원통 투영 + 7파라미터
  (높이, el_top/bot, 요 오프셋, 헤딩, 필드 내 카메라 위치).
  LM 감쇠 Gauss-Newton, 수치 야코비안, numpy 만.
- **휴리스틱 자동 매칭**: 랜드마크를 하나하나 고르지 않는다 —
  "랜드마크 찍기" 모드에서 클릭하면 기본(또는 현재 피팅) 카메라 모델로
  지면 좌표를 추정해 최근접 미지정 랜드마크에 배정. 합성 검증:
  기본 가정과 꽤 다른 카메라(5.5m, 8m 치우침, 회전)에서도 8/8 정답.
  틀리면 우클릭 "이 위치를 랜드마크로 지정"으로 수정, 재클릭=이동.
- 4점 이상 되면 자동 피팅 → 상태 라벨에 잔차·카메라 높이·터치라인
  거리, 미리보기에 **예상 경기장 선**(외곽·중앙선·센터서클) 오버레이로
  피팅 품질을 눈으로 확인.
- 레이더가 경기장 절대 좌표 모드로 전환: 실측 경기장(외곽/중앙선/
  센터서클/카메라 위치)을 그리고 pano_to_field 로 선수·공 배치.
- `.ptz.json` 에 field_points/field_size 저장. 경기장 크기 조정 가능
  (기본 105×68).
- 테스트: 합성 카메라 라운드트립(<1m), 4점 최소, 3점 거부, 클릭 노이즈
  σ4px에서 rms<10px, 역변환 일관성.

## NLE 스타일 멀티트랙 타임라인 (TimelineView)

기존 단일 트랙바를 편집 프로그램식 타임라인으로 교체:

- 레인 5개: **크롭/KF**(위치=노랑, 크롭 폭 포함=주황), **공**(수락=초록,
  무시=빨강, 승격=자홍 틱), **팀1 / 팀2**(GK 포함, 역할 색),
  **기타**(심판·미분류). 선수 트랙릿은 겹침 회피 서브행 3개에 배치.
- **Ctrl+휠 = 커서 기준 확대축소**(최대 30px/프레임), 휠/빈 곳 드래그 =
  팬, 빈 곳 클릭 = 시크, 눈금자는 줌에 맞춰 1s~10min 단위 자동 선택,
  빨간 플레이헤드.
- 바 클릭 = 선택(흰 테두리) + 해당 목록(공/오인식/선수) 선택 연동 →
  프레임 이동. 선수 목록 선택도 타임라인에 하이라이트 역연동.

## 분석 메뉴 (사용자 편집 초기화)

메뉴바에 "분석" 추가 — 편집을 무효화하고 **분석 원본으로 되돌리기**:
공/키프레임 편집(키프레임·무시·승격) / 선수 역할 / 경기장 캘리브레이션 /
모든 사용자 편집. 삭제 대상 개수를 보여주고 확인 후 실행. 분석 자체
(.analysis.json)는 검수로 안 바뀌므로 에디트만 지우면 곧 원본이다.
