## 검수 UX (사용자 피드백 연속 반영)

- **목록 의미 재편**: 위 = 공(자동 트랙 + "● 수동" 키프레임 병합, 시간순),
  아래 = 오인식(공 아님). 하단 가로 스트립 배치(공|오인식|레이더),
  영상은 전체 폭.
- **키보드 검수**: ↑↓ 선택=이동, →/Del=오인식으로, ←=복원. 무시 후
  다음 항목 자동 선택. 트랙바(초록/빨강/주황) 슬라이더와 폭 정렬.
- **반응성**: 링크·선수통계 캐시 분리(LinkWorker)로 계획 재계산
  3.6s→0.13s, 프레임 캐시 리드로우로 클릭 반영 ~50ms. 미끼 일괄
  무시(같은 자리 정적 트랙 수집). 취소 검출은 빨간 X "IGNORED".
- **스크럽 프록시**: 1920px·g15 사본(NVENC, 완주 883MB) → 시크
  810ms→46ms. 생성 진행률 표시(-progress 파싱).
- 원경 줌 설정(far_zoom, 기본 1.0), 근경 가변 줌(near_widen 1.6).

## 선수 트래킹·팀·레이더

- 분석이 ByteTrack(track 모드, lapx 필요)로 선수 ID 부여,
  [cx,cy,w,h,id,상반신HSV] 저장. classify_teams: 채도가중 색상 k-means
  (상위 2군집=팀, 나머지=심판/GK). 팀 색 박스 오버레이 + 탑다운 레이더
  (원통 기하 지면 투영, 카메라 기준 m, 10m 격자). 공 박스도 저장
  (학습 라벨용). 오인식 마킹은 비파괴 — export_training_labels 로
  하드 네거티브/양성 라벨 추출 가능.

## 검출 강화

- **모델 선택 설정**: yolov8n/s/m, yolo11n/s/m, 사용자 .pt (QSettings).
  CUDA 추론 확인(2080 Ti) — 병목은 디코딩이라 큰 모델 부담 적음.
- **far_boost**: 원경 밴드(field_top~58%)를 **정사각 타일 ~12장**(밴드
  높이≈540 ≈ 학습 크기 640, 15% 겹침, 배치 추론)으로 공 전용 네이티브
  검출. 실측: 원경 공 0/21 → 6/21. (와이드 타일 2장은 4/21 — 정사각이
  학습 분포 정합으로 우위)
- **다중 후보**: 샘플당 상위 3개 공 후보 저장(ball_cands) + 후보 집합
  위의 트랙 연결 — 미끼가 conf 로 이겨도 진짜 공 트랙이 공존.
  **무시에 위치 추가**(f0,f1,x,y): 그 자리(150px) 트랙만 기각 — 같은
  시간대의 진짜 공 보호 (테스트로 검증). 구형 항목/캐시 호환.
