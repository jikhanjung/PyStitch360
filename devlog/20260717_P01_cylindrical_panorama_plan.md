# P01: 원통 파노라마 기반 축구 경기 영상 스티칭 계획

- 날짜: 2026-07-17
- 종류: 계획 (P)
- 상태: 프로토타입 검증 완료

## 1. 배경과 문제 정의

같은 목표(듀얼 GoPro로 축구장 전체 영상 제작)를 두 가지 방식으로 시도했다:

- **PitchStitch** (`../PitchStitch`): 수동 랜드마크 4점 → 평면 원근 변환 → 좌우 hstack.
  평면(rectilinear) 투영은 합산 화각이 90°를 넘으면 가장자리 확대가 발산하는 본질적
  한계가 있어, 가까운 쪽이 지나치게 커지고 먼 쪽이 작아짐.
- **PyStitch360** (구 버전, `legacy/`): 360° equirectangular 출력. 실제 콘텐츠는
  전방 절반뿐이라 360 플레이어 시청이 오히려 불편. 프레임마다 cv2.Stitcher 재정합
  (심 흔들림) + 픽셀별 파이썬 루프(성능)라는 구현 문제도 있었음.

**결론: 두 방식의 중간인 원통(cylindrical) 투영 파노라마가 정답.**
수평 방향 확대율이 균일하고 발산이 없으며, Veo/Pixellot 등 상용 스포츠 중계
시스템이 쓰는 방식과 동일한 접근.

## 2. 촬영 조건 (확정된 제약)

- 카메라: GoPro HERO5 Black × 2, 4K 16:9 29.97fps, **Wide 모드, EIS 끔** (필수)
- 설치: 하프라인 부근 터치라인에서 3~5m, 삼각대 (높이 ~2m, 낮음)
- 리그는 비고정 — 세션마다 방향이 다름. 단, 두 카메라는 한 바에 최대한 밀착
  (근거리 시차 최소화; 광축 간격 목표 ~10cm 이내)
- 촬영 중 각도는 거의 고정. 바람/공 충격으로 간혹 틀어질 수 있음 → 구간별
  고정(piecewise-constant) 방향 모델로 대응
- 근거리(심 정면 3~5m) 시차로 인한 선수 이중상은 **수용하기로 결정** (물리적으로
  제거 불가, 페더 블렌딩으로 완화만)

## 3. 파이프라인 아키텍처

```
입력: 좌/우 GoPro 챕터 파일들 (GOPRxxxx + GP01xxxx...)
  1. 챕터 연결        — 파일들을 하나의 연속 타임라인으로 취급
  2. 동기화           — 오디오 상호상관 자동 추정 + 수동 미세조정
  3. 렌즈 왜곡 보정    — 내장 렌즈 프로파일 (OpenCV fisheye 모델)
  4. 세션 캘리브레이션 — 경기장 랜드마크로 카메라 방향 추정
  5. 원통 워핑        — remap 테이블 사전 계산, 프레임당 cv2.remap 2회
  6. 게인 보정 + 페더 블렌딩
  7. 출력             — (a) 초광폭 파노라마 (~5,500×1,500, HEVC 권장)
                        (b) 가상 PTZ 크롭 (공 추적 16:9, 추후)
```

핵심 원칙: **모든 기하 계산은 시작 시 1회 (remap 테이블 캐싱), 프레임 루프에는
벡터화된 remap/블렌딩만.** 고정 리그가 아니어도 세션 내에서는 방향이 고정이므로
프레임별 재정합은 하지 않는다.

## 4. 캘리브레이션 전략 (2층 구조)

| 파라미터 | 갱신 주기 | 방법 |
|---|---|---|
| 렌즈 왜곡·초점거리 | 카메라당 1회 | Gyroflow 렌즈 프로파일 DB (내장 프리셋) |
| 카메라 방향 (초기) | 세션당 1회 | 겹침 영역 특징점 매칭 (자동) + 경기장 랜드마크 (수평/검증) |
| 카메라 방향 (변동) | 충격 이벤트당 | 앵커 추적 or GPMF 자이로 → 재추정, 새 세그먼트 |
| 동기화 오프셋 | 세션당 1회 | 오디오 상호상관 (자동) |

- 렌즈 프로파일: `presets/lens_profiles/GoPro_HERO5_Black_Wide_4K_16x9.json`
  (Gyroflow lens_profiles DB, OpenCV fisheye 모델, 4K 16:9 Wide NO-EIS 전용)
- 경기장 랜드마크: 센터서클(r=9.15m)·페널티박스(40.32×16.5m)는 규칙상 크기 고정
  → 실측 없이 미터 기준 확보. 경기장 전체 크기는 가변이므로 프로젝트 설정에 입력 필드.
- HERO5 Black은 GPMF 텔레메트리(자이로/가속도) 내장 → 충격 이벤트 감지와 수평
  보정 보조에 활용 가능 (gpmf-parser 계열 라이브러리).

## 5. 프로토타입 검증 결과 (2026-07-17)

`prototype/stitch_still.py` — 실제 경기 영상
(`/mnt/f/Pictures/20260712_GoPro5_L,R`)으로 검증:

- **오디오 동기화**: 상호상관 피크/잡음비 26배로 명확. 이번 세션 오프셋
  **-0.068초 (~2프레임, 우측이 먼저 시작)**. 세션마다 다를 수 있으니 자동 측정 필수.
- **자동 방향 추정**: 겹침 영역 SIFT 매칭 1,206쌍 → 광선 변환 → RANSAC+Kabsch.
  인라이어 857개, 잔차 중앙값 0.127°. 상대 회전 68.8°.
  경기장 랜드마크 없이도 특징점만으로 정합 성공 (랜드마크는 수평 보정·미터 기준용).
- **원통 파노라마**: 5,907×1,680 (수평 193°), 심이 잔디에서 식별 불가 수준.
  하프라인이 심을 가로질러 연속. 게인 보정으로 좌우 색감 차이(적색 채널 ~8%) 해소.
- **수평 보정**: 카메라 평균 방향이 ~18° 아래를 향해 pitch 보정 필요했음 (수동 18°).
  → 랜드마크 기반 자동 수평 추정이 필요한 이유. 잔여 roll 오차 약간 남음 (좌측이 살짝 높음).

버그 이력: 절반 회전 배정 방향 (R_wl=Rh, R_wr=Rh.T가 맞음. 반대로 하면 겹침이
바깥으로 뒤집힌 이중상 파노라마가 됨).

## 6. GUI 계획 (PyQt6, 새로 작성)

기존 코드는 `legacy/`로 이동, 참고만 한다. 새 GUI 요구사항:

1. **듀얼 뷰어 + 동기화**: 좌/우 영상을 나란히 표시, 프레임 단위 이동,
   오디오 자동 동기화 버튼 + 수동 오프셋 조정
2. **챕터 통합**: GoPro 분할 파일(GOPR→GP01→GP02...)을 자동 그룹핑해서
   하나의 연속 영상처럼 탐색/처리
3. **캘리브레이션 탭**: 프레임 선택 → (자동 특징점 정합) → 랜드마크 클릭(수평/검증)
   → 파노라마 미리보기 → pitch/roll 미세조정
4. **세그먼트 관리**: 타임라인에 방향 변동 이벤트 표시, 세그먼트별 재캘리브레이션
5. **출력 탭**: 해상도/코덱(H.264/HEVC)/CRF, 파노라마 or 가상 PTZ 선택
