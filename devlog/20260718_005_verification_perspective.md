## 완주 파노라마 검증 (39.5분, 7.2GB)

어제 완료된 `match_20260712_pano.mp4`를 검증:

- **챕터 경계 연속성 (4곳)**: 경계를 건너는 프레임 diff가 주변 프레임 diff
  중앙값과 동일 수준 (0.70~1.10 vs 0.63~0.95) — 점프/끊김 없음. 경계 전후
  프레임 육안 확인도 정상 (미세한 노출 차이만).
- **A/V 동기 (원본 L 오디오와 교차상관, 챕터별 5지점)**: lag
  0 / +12 / +12 / +24 / +24ms, 상관계수 1.00. 최대 지연도 1프레임(33ms)
  미만이라 합격. 다만 챕터가 넘어갈수록 +12ms씩 누적되는 경향이 있어
  더 긴 세션(2시간+)에서는 챕터 경계 오디오 타임스탬프 처리를 재점검할 것.

## 원근비 조절 (`core/perspective.py`)

요청: 파노라마에서 가까운 곳은 zoom out, 먼 곳은 zoom in — 터치라인 촬영의
근/원경 배율 격차 완화. 정지 프레임으로 5개 변형(A~E)을 비교한 결과
**E안 확정: 키스톤 m=1.3 + 수직 리맵 k=0.3**.

- 수직 리맵: 수평선 아래 정규화 좌표 u에 대해 소스 행 f(u) = (1-k)u + ku².
  수평선 직하 세로 배율 1/(1-k), 최하단 압축 (1+k). 내용 손실 없음.
  단, 원경이 세로로만 커져서 강하게 걸면(k=0.6) 선수가 길쭉해짐.
- 키스톤(사영 기울임): 행 스케일 w(y)로 가로·세로 동시 확대(최상단 m배).
  상단 좌우가 (1−1/m)만큼 잘리지만 그 영역은 트랙/여백이라 무해.
- 두 변형 모두 행 단위 스케일이라 **하나의 cv2.remap 맵으로 합성** (보간
  1회). 합성 맵 == 순차 적용 결과임을 테스트로 확인.
- 순수 함수 이슈 하나: 최하단 행 고정점을 h가 아니라 **h−1**로 잡아야
  끝행이 정확히 항등이 됨 (초기 구현은 0.5px 오프셋 발생, 테스트로 검출).

`scripts/pano_perspective.py`: 완성 파노라마에 재스티칭 없이 적용
(리맵+재인코딩, 오디오 스트림 복사). `--k --m --horizon` 노출.
수평선 기본값은 높이의 23% (이번 정합에서 실측 380px/1680px).
렌더 파이프라인에 통합할 때는 elevation=0 행을 쓰면 정확 —
내보내기 옵션으로 GUI 노출은 추후.

유닛 테스트 6개 추가 (`tests/test_perspective.py`): 항등/단조/배율/합성
일치/파라미터 검증. 전체 18개 통과.
