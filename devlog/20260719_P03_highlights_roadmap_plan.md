# P03: 하이라이트 → 리포트 로드맵 계획

- 날짜: 2026-07-19
- 종류: 계획
- 관련: [P02](20260718_P02_virtual_ptz_plan.md), [022](20260719_022_kickoff_events.md)~[026](20260719_026_ui_responsiveness.md)
- 순서: 1 → 6 차례 진행 (사용자 결정)

## 배경

검출·이벤트 인프라가 갖춰짐: 킥오프(.events.json), 확실한 호각 23개,
선심 기 신호(파울/오프사이드), 뜬 공 구간(공중 탄도), 사용자 이벤트,
경기장 절대 좌표(캘리브레이션), 팀/역할 분류. 이제 이것들을 **산출물**
(공유 가능한 클립·오버레이·리포트)로 잇는다.

## 1. 하이라이트 자동 추출 + 일괄 클립 내보내기

**목표**: 이벤트 융합으로 하이라이트 후보 구간을 제안하고, 검수 후
선택 구간들을 한 번에 개별 클립으로 내보낸다.

### 후보 생성 (core/highlights.py)

이벤트 소스별 규칙 → (t0, t1, 종류, 점수):

| 규칙 | 구간 | 근거 |
|---|---|---|
| 킥오프 | [t−10s, t+20s] | 경기 시작/득점 재개 |
| 기 신호(foul/offside) | [호각−8s, +7s] | 파울·오프사이드 장면 |
| 롱 휘슬(≥0.8s, 비킥오프) | [t−8s, +5s] | 주요 판정 |
| 뜬 공 → 페널티박스 방향 | [발사−4s, 착지+6s] | 슛/크로스 후보 |
| 공 속도 급증 (>20m/s) | [t−3s, +5s] | 강슛/롱패스 |
| 사용자 이벤트 | [t−8s, +8s] | 수동 마크 (골 등) |

- 겹침 병합(간격 <3s), 점수 = 규칙 가중 합 (사용자 이벤트 최상).
- `.events.json` "highlights" 저장 (수락/제외 상태 포함 — 비파괴).

### GUI

- 분석 메뉴 "하이라이트 후보 생성" → 이벤트 레인에 구간 바(호박색)로
  표시, 클릭=이동, 우클릭=수락/제외/경계 조정(IN/OUT 재사용).
- **일괄 내보내기 대화창**: 수락 구간 목록(시각·길이·종류·점수),
  체크박스, 출력 폴더, 코덱/CRF/미니맵 옵션(기존 ExportDialog 요소
  재사용) → 구간별 `<이름>_h01_킥오프.mp4` 순차 렌더 (render_plan
  start/end 재사용, 진행률 = 전체 구간 합).

### 완료 기준

실경기에서 후보 ~15±10개 생성, 육안 확인으로 과반이 유의미, 3개 구간
일괄 내보내기 동작 (오디오 싱크 포함).

## 2. 경기 시계 오버레이

**목표**: 내보내기 영상 좌상단에 "전반 12:34" 시계.

- 앵커: 킥오프 이벤트(자동) 또는 사용자 지정. 후반 영상은 앵커 라벨만
  "후반"으로 (+45:00 옵션).
- render_plan 에 clock 파라미터: 프레임 → 경기 시각 문자열, 반투명
  배경 박스 + 흰 글자 (cv2.putText, 미니맵과 동일한 스케일 규칙).
- ExportDialog 에 "경기 시계" 체크박스 + 앵커 선택 콤보(킥오프 목록).
- 팀 이름 입력돼 있으면 "A 0 : 0 B" 스코어 자리도 (스코어는 사용자
  이벤트 '골' 카운트 — 없으면 시계만).

## 3. 트랙릿 병합 (ReID 라이트)

**목표**: 2,135 트랙릿 → 사람 수십 단위로 병합. ML ReID 전 단계.

- 병합 조건: (a) 시간 겹침 없음, (b) 끝↔시작 시공간 근접 (Δt<3s 이고
  필드 거리 < Δt×8m/s), (c) 유니폼 색 거리(classify_teams 특징 공간)
  < 문턱, (d) 역할 일치(사용자 오버라이드 우선).
- 그리디 체인 병합 → 병합 맵 {tid: 대표 tid} 을 `.ptz.json`
  "merges" 저장 (비파괴 — 분석 원본 불변). GUI 는 표시·통계에서 대표
  tid 사용, 선수 목록에 병합 그룹 표시 + 수동 병합/분리 메뉴.
- 완료 기준: 실경기 트랙릿 수 1/5 이하로, 선심 커버 공백 감소 확인,
  잘못 병합(다른 사람 연결) 육안 표본에서 드묾.

## 4. 선수 히트맵/활동량 리포트

**목표**: 경기 후 리포트 이미지/HTML.

- 팀별 히트맵(경기장 비율 그대로, 등방), 선수(병합 tid)별 히트맵·
  이동거리·평균/최대 속도 표. 3번 병합 품질에 의존 — 순서 유지 이유.
- scripts/report.py → PNG 묶음 + 요약 markdown. GUI 는 분석 메뉴 항목.

## 5. 자체 파인튜닝 파이프라인

**목표**: 이 경기장 특화 공 검출 recall 개선 (원경 중심).

- export_training_labels 확장: 수락 트랙=양성, 무시=하드 네거티브,
  갭필/시드 주입=양성(저신뢰 태그). 타일 크롭 데이터셋(640) 생성.
- yolo11m 파인튜닝 (2080 Ti, 짧은 epoch) → A/B 스크립트(020 재사용)로
  원경 검출률 전후 비교. 개선 시 기본 모델로 등록.

## 6. 후반 영상 검증 (사용자 주도 + 보조)

- 후반 스티칭 → 분석 → 호각/이벤트 파이프라인 실행.
- 검증 항목: 후반 킥오프 검출(진영 교대 무가정 확인), **득점 역추론**
  (경기 중 킥오프 = 직전 골 → 사용자 이벤트 자동 제안), 오프사이드
  기 신호 실사례, 하프 간 트랙릿/팀 색 일관성.
- 여기서 나온 이슈가 1~5 의 파라미터 튜닝으로 되돌아옴.
