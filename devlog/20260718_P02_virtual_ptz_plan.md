# P02: 가상 PTZ 종합 계획 (자동 검출 기반 + 수동 키프레임)

- 날짜: 2026-07-18
- 종류: 계획
- 관련: [007 PTZ v2](20260718_007_ptz_v2_two_pass.md),
  [003 v1 구현](20260717_003_project_segments_ptz.md)

## 목표

완성된 원통 파노라마(원본판 — 원근 보정판은 화질 손해로 PTZ 소스에서 제외)
에서 공/경기 흐름을 따라가는 1920×1080 가상 카메라 영상을 만든다.
자동 추적만으로는 신뢰도가 부족하므로 **자동 검출을 기본 궤적으로 깔고,
사용자가 GUI 에서 공 위치를 클릭(키프레임)해 덮어쓰는** 반자동 구조.

## 지금까지의 경위와 확정 사항

1. **v1 (실시간 스트리밍 추적)** — 실패로 판정.
   - 실효 검출 해상도 960px (imgsz 불일치 버그) → 공 검출률 **0%**.
   - 공 부재 시 "선수 전체 가중평균" 추종 → 검출 깜빡임마다 좌우 흔들림.
2. **검출률 실측** (경기 전체 58 샘플, 2944px):
   경기장 안 공 conf≥0.25 **52%**, 확실(≥0.4) 29%, 미검출 48%.
   시간축 보간을 더하면 실효는 더 높지만 완전 자동은 무리 — **사용자 결정:
   수동 키프레임이 주 경로, 자동은 기본값/보조.**
3. **v2 (2패스 오프라인)** — 현재 코어 (`core/ptz.py`):
   - 1패스 `analyze_video`: 3프레임마다 2944px 검출, 장외(상단 26%) 필터,
     `.analysis.json` 캐시 (파라미터 반복이 초 단위).
   - `build_plan`: 공 후보를 **트랙**(시간 연속 연결, 최근 5점 평균 기준,
     연결 거리 상한 850px)으로 묶고 **트랙 단위로 기각** — 사용자 지적
     ("프레임마다 수동으로 지울 수 없다") 반영. 기각 규칙: ① 정지+고립
     (r80≤30px & 고립>50% & 3s+ — 낙엽), ② 5s+ 지속 트랙의 고립>50%
     (장외 공 — 실측상 진짜 공 트랙 고립률 0~4% vs 미끼 ~80%로 명확 분리),
     ③ 수락 트랙과 시간 중복 50%+ (경합 오검출). 이후 2초 갭 보간 →
     공 부재 시 선수 분포 커버 **줌아웃**(최대 (1680−160)/1080 ≈ 1.4배) →
     zero-phase 가우시안 스무딩 (기본 σ=1.2s, 잔차 400px+ 구간만 σ=0.35s
     빠른 추종 크로스페이드). 상단 검은 스티칭 경계 회피 top_margin=160px.
   - `render_plan`: 2패스 크롭/다운스케일 + 인코딩, 오디오 복사.
   - 사용자 방침: 경기장 밖 공 무시(장외 필터), 공 불확실 시 선수를
     최대한 담는 줌아웃, 공이 빠를 때만 빠른 카메라 — 반영됨.
4. **키프레임 병합** (오늘 추가): `build_plan(keyframes=[(frame,x,y),...])`
   - 키프레임 ±1.5s 의 자동 샘플은 무효화 (자동이 수동을 되끌지 않게).
   - 인접 키프레임 쌍은 8s 까지 직접 보간으로 연결 (자동 억제 구간 커버).
   - 키프레임은 게이팅/미끼 필터를 우회 (항상 신뢰).

## GUI 설계 (4번 탭 "가상 PTZ")

- **파노라마 열기**: 완성 mp4. 열기 대화상자는 현재/최근 영상 폴더에서 시작.
- **자동 분석**: 백그라운드 (완주 ~20분, 진행 로그). `<pano>.analysis.json`
  캐시 자동 재사용.
- **미리보기 + 타임라인**: 슬라이더/스텝, 현재 프레임에 자동 공 위치(원)와
  키프레임(십자) 오버레이. 계획된 크롭 창 사각형 표시 (분석 후).
- **키프레임 입력**: 화면 클릭 = 현재 시각에 공 위치 지정. 목록에서
  이동/삭제. `<pano>.ptz_keyframes.json` 사이드카에 자동 저장.
- **PTZ 내보내기**: build_plan(키프레임 포함) + render_plan 백그라운드,
  진행바 + 주기 로그.
- 워크플로우: 자동 분석 → 영상 훑으며 "공이 아닌 곳을 보는 구간"만
  클릭으로 교정 → 내보내기. 검출률 52% 기준, 교정 필요 구간은 경기의
  일부일 것으로 예상.

## 향후 (로드맵, 사용자 요청 순)

1. **킥 동작/선수 움직임 기반 카메라 예측** — 포즈/모션 큐로 패스 방향을
   선행 추적 (사람 카메라맨처럼).
2. **선수 전원 개별 추적** — ID 유지 (ultralytics track / ByteTrack 계열).
   PitchAnalysis 계열 분석(히트맵, 이동거리)의 기반.
3. 공 검출 자체 개선 후보: 파노라마 특화 파인튜닝, 모션 큐(빠른 소형 물체),
   타일 추론.

## 실행 환경 메모

내보내기 실행은 Windows 네이티브(NVENC, RTX 2080 Ti) 예정 — WSL2 는 NVENC
미지원. 프로젝트/분석 파일은 WSL↔Windows 경로 자동 변환 지원.
