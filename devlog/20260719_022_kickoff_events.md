## 킥오프 검출 (core/events.py)

호각 × 대형(포메이션) 융합 — 규칙 8조가 강제하는 배치를 기하로 읽음:

- `formation_track`: 샘플별 {분리도 sep, 센터서클 인원, 공 센터 거리,
  팀 인원}. sep = min(팀별 자기 진영 비율), 진영은 팀 X 중앙값 부호
  (전/후반 교대 무가정). GK(3/4)는 팀 0/1 로 접음, id<0 은 서클 인원만.
- `detect_kickoffs`: 호각(≥15dB)마다 pre[t-12s] 분리도 중앙값 ≥0.8 &
  서클 ≤4명, post[t+20s] 분리도 ≤0.55(대형 붕괴) 또는 공 8m+ 이탈.
  30s 내 중복은 최고 점수. → `.events.json`.
- CLI scripts/events.py, GUI 분석 메뉴 "킥오프 검출".

## 실데이터 검증 — 검출기가 사람의 가정을 바로잡음

킥오프 1개 검출: **4:36.7** (분리도 0.88, 서클 1명, 대형붕괴+공이탈).
0:17.3 롱 휘슬(devlog 021 에서 킥오프로 추정)은 기각 — 프레임 확인
결과 **영상 처음 ~30초는 카메라 설치 중**(t=2s 손가락, 12~32s 빈
필드)이라 경기 전 소음이 맞다. 4:36 킥오프 + 39:17~39:31 종료 호각
= 약 35분 하프로 정합. 대형 조건이 경기 전/파울 호각 92개를 전부
걸러냈다 (합성 테스트 3종 통과).

## 이벤트 트랙 (타임라인 7번째 레인)

- 자동(킥오프)=초록, 사용자=시안 마커 + 라벨. 클릭=이동.
- 우클릭 메뉴: "여기에 이벤트 추가..."(이름 입력), 근처(±3s) 사용자
  이벤트 삭제. `.ptz.json` "user_events" 저장. 자동 이벤트는 읽기 전용.

## 팀 이름 + UI 다듬기

- **팀 이름 입력**: 선수 탭 "팀 이름..." 버튼 → 대화상자. 역할 표시명
  전체 교체(_role_name): 범례·역할 버튼·선수 목록·**우클릭 메뉴**·
  타임라인 팀 레인 라벨. `.ptz.json` "team_names" 저장.
- 타임라인 레인 제목 **세로 중앙 정렬** (기존: 바닥 붙음).
- **하단 가장자리 드래그 = 전체 높이** 비례 조절 (Shift 불필요,
  레이블 영역 경계 드래그 = 개별 레인은 유지).
- 선수 목록에서 **GK 는 하단 그룹**으로 (그 안에선 검출 수 순).
