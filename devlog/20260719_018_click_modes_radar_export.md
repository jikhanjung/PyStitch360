## 빈 곳 클릭 모드: 공(기본) / 키프레임

- 데이터 모델은 기존 keyframes 그대로 — **3요소 = 수동 공**(위치만,
  줌 자동), **4요소 = 크롭 키프레임**(위치+폭 앵커). 사이드카 호환 유지.
- 트랜스포트에 **공 / KF 토글 버튼** (기본 공). 공 모드 빈 곳 클릭 =
  3요소, KF 모드 = 현재 계획 폭을 얹은 4요소.
- **우클릭 전환**: 마크 우클릭 → "공으로 전환(줌 자동)" ↔ "크롭
  키프레임으로 전환(현재 폭 고정)". 빈 곳 우클릭 메뉴에도 두 종류 추가.
- 표시 구분: 수동 공 = 주황 이중 링, 키프레임 = 빨강 4방향 화살표.
  목록 라벨도 "● 수동 공" / "◆ 키프레임". (타임라인 레인 0 은 기존
  노랑/주황 구분 그대로)

## 첫 프레임 점프 수정 (검수 중)

트랙을 따라가며 승격+무시하면 영상 처음으로 튀는 버그 — 스택 추적으로
범인 확정: 무시 후 "다음 항목 자동 선택"(키보드 검수용)이
currentRowChanged → _goto_track 시크를 유발, 다음 항목이 영상 초반
트랙/마크면 처음으로 이동했다. 수정:

- 자동 선택은 **시그널 차단 상태로** (시크 없음). 다음 항목으로의
  이동은 **키보드 검수(Del/→)에서만** (advance 플래그).
- _ignore_selected_track 의 "무시 전 트랙 시작으로 시크" 제거 —
  anchor_f 파라미터로 대상 트랙을 지정 (재생 위치 불변).
- _goto_track: 슬라이더가 이미 그 트랙 구간 안이면 시작으로 되감지
  않음 (따라가던 문맥 유지).
- 공 목록도 재구성 시 선택 보존 (수동 마크는 프레임, 트랙은 현재
  위치 포함 구간으로 매칭) — 선수 목록과 동일 패턴.

## 내보내기 레이더 오버레이 (옵션)

- **core**: build_radar_data(분석+역할+캘리브레이션 → 샘플별 필드 좌표
  선수/공) + draw_radar_panel(등방 축척 탑다운 패널 — **경기장 사각형이
  입력 크기 비율 그대로**, 이 경기장은 100×62m). render_plan 에
  radar 파라미터: 우하단 반투명(α 0.55) 합성, 샘플 단위 패널 캐시.
- **GUI**: 내보내기 바에 "레이더" 체크박스 (QSettings 저장, 기본 켬).
  체크 시 역할 오버라이드·유니폼 색 팔레트·field_size·수동 검출 포함해
  데이터 조립. 캘리브레이션 없으면 카메라 기준 근사 좌표로 동작.
- 실측 검증: 중반 프레임에서 점 배치가 실제 장면과 일치 (렌더 확인).

## 유니폼 색 컬러피커

선수 탭 범례의 색 칩이 버튼으로 — **클릭 = QColorDialog** 로 역할
표시 색 지정(박스·레이더·내보내기 팔레트 즉시 반영, 라벨에 ✎),
우클릭 = 측정색 복귀. 우선순위: 사용자 지정 > 측정 대표색 > 기본
팔레트. `.ptz.json` "kit_colors" 저장, 역할 초기화에 포함.
