## scripts/jersey_ocr.py

- **게이트**: 필드 Y<0 (근측 절반) + 박스 높이 ≥90px 검출만 시도 —
  원경은 수십 px 라 인식 불가라는 방침 그대로.
- 트랙릿(병합 대표)당 큰 박스 순 최대 12장, 2초 이상 간격으로 분산
  선택 → 상반신 크롭(폭 ±0.65w, 높이 −0.55h~+0.15h) 2배 업스케일 →
  easyocr 숫자 전용(allowlist) 인식.
- **신뢰도 가중 투표**: 트랙릿별 1~2자리 숫자 후보를 conf 합산으로
  집계, 문턱(votes×conf) 미달은 버림. 기존 지정(player_nums)과
  일치/충돌 표시.
- 결과는 .events.json "ocr_numbers" 에 **제안으로만** 저장 (비파괴).

## GUI 연동

번호 지정 메뉴에 "OCR 제안: 7번 (지분 75%)" 항목이 뜨고 클릭하면
바로 지정 — 적용 여부는 항상 사용자.

## 실행 (Windows)

```
pip install easyocr          # torch 는 ultralytics 환경에 이미 있음
python scripts/jersey_ocr.py <pano.mp4> [--gpu]
```

검증 포인트: 근측 게이트를 통과한 트랙릿 수, 제안 정확도(기존 수동
지정과 일치율), 충돌 건. 등번호는 등을 보일 때만 보이므로 지분(share)
이 낮아도 최고 득표가 맞을 수 있음 — 문턱은 결과 보고 조정.
