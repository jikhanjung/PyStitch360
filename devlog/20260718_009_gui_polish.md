# 009: GUI 실사용 피드백 배치 (내보내기 크래시, 최근 프로젝트, UX)

- 날짜: 2026-07-18
- 종류: 작업 기록
- 관련: [008](20260718_008_align_tab_timeline_playback.md)

사용자가 WSLg 로 GUI 를 실사용하며 준 피드백/버그를 일괄 처리.

## 버그

- **내보내기 즉사 (SIGABRT)**: `ExportWorker` 가 시작 시각을 `self.start` 에
  저장해 **QThread.start() 메서드를 float 으로 가림** → GUI 버튼 경로 첫
  실행에서 `TypeError: 'float' object is not callable`. 어제 완주 내보내기는
  헤드리스로 `_run()` 직접 호출이라 드러나지 않았음. `t_start/t_end` 로
  개명 + 워커 속성이 QThread API 를 가리지 않는지 회귀 테스트.
- **GUI 를 `| head -20` 으로 띄운 세션 사고**: 출력 20줄 초과 시 SIGPIPE 로
  앱 전체 종료. 실행은 로그 파일 리다이렉트로 (재발 방지: 파이프에 GUI 앱
  절대 금지).
- 정합 실패 게이트: 인라이어 <30 이면 명확한 오류로 거부 (실사례 3/108
  인라이어 → auto-level ±25° 발산 → 검은 화면). compute_gains 0 나눗셈 가드.

## 기능/UX (사용자 요청)

- **최근 프로젝트 메뉴** (QSettings, 최대 10개, 소실 파일 자동 제외, 목록
  비우기). 열기/저장 시 갱신.
- **자동 정합값 상시 표시**: 미세조정 그룹에 현재 세그먼트의 자동
  pitch/roll/yaw 를 회색 라벨로 (이 경기: pitch +28.5°, roll −5.7°,
  yaw −4.5°). 슬라이더 이동으로 세그먼트가 바뀌면 따라 갱신.
- **재정합 시 수평/센터링 재사용**: 한 경기에서 수평이 바뀌는 일은 거의
  없으므로(사용자 지적) 세그먼트 추가 정합은 기존 pitch/roll/yaw 를
  그대로 쓰고 상대 회전만 재추정 — 세그먼트 간 뷰 흔들림 제거 + 정합
  최다 소요 단계(auto-level) 생략.
- **정합 후 자동 재생 제거** (008 에서 넣었던 것) — 정지 미리보기 갱신만,
  재생은 ▶ 버튼.
- 미세조정 범위 ±15° → ±45°.
- **wait cursor**: 블로킹 구간(프로젝트/영상 열기, 프레임 디코딩)과
  백그라운드 대기(싱크·정합·자이로·재생 준비 첫 프레임까지)에
  setOverrideCursor 쌍 삽입. offscreen 왕복 테스트로 스택 균형 확인.

## 방향 확정 (사용자)

- 원근 보정판(4.75GB, 완료)은 화질 손해 감수용 — **PTZ 는 원본 파노라마
  기반으로 계속 진행**. 다음 단계는 GUI 공 위치 수동 키프레임.
