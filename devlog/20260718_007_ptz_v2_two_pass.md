# 007: PTZ v2 — 2패스 트래킹, 공 검출률 실측, 향후 방향 결정

- 날짜: 2026-07-18
- 종류: 작업 기록
- 관련: [005](20260718_005_verification_perspective.md)

## 문제 (사용자 피드백)

v1 완주 PTZ: 볼 트래킹 실패 잦음 + 좌우 흔들림으로 시청 곤란.

## 원인 실측

- **v1 설정의 공 검출률 = 0%.** 5906px 파노라마를 1280px 로 축소한 것을
  YOLO 가 imgsz=960 으로 재축소 → 공이 ~2px. (Detector imgsz 와 축소폭
  불일치 버그 — 수정함)
- 검출 해상도 2944px: 경기장 안 공 검출 52% (conf≥0.25), 확실(≥0.4) 29%.
  경기 전체 40초 간격 58프레임 실측, ~47ms/f.
- 흔들림: 공 부재 시 "선수 전체 가중평균" 추종 — 검출 깜빡임마다 무게중심
  점프 + 어쩌다 잡힌 공(30배 가중)으로 순간이동.

## v2: 2패스 오프라인 설계 (`core/ptz.py`)

1. **1패스 analyze_video**: 3프레임마다 2944px 검출, 장외(상단 26%) 필터.
   결과는 `.analysis.json` 캐시 — 스무딩 파라미터 반복 조정이 초 단위.
2. **build_plan**: 공 게이팅(순간이동 기각) → 2초 갭 보간 → 공 부재 시
   선수 트림 평균 + 분포 커버 줌아웃(최대 (1680-160px)/1080 ≈ 1.4배) →
   zero-phase 가우시안 스무딩 (기본 σ=1.2s, 잔차 400px+ 구간만 σ=0.35s
   빠른 추종 크로스페이드).
3. **정적 미끼 필터**: 3분 샘플에서 낙엽이 conf 0.45~0.65 '공'으로 수십 초
   검출되어 빈 코너를 응시하는 문제 발견. 시그니처 = **장시간 정지(±30px)
   + 모든 선수로부터 700px+ 고립** → 기각. 선수 검출이 없는 샘플은 고립
   판정 불가로 보고 유지. 샘플에서 미끼 기각 16%.
4. **render_plan**: 계획대로 크롭(줌아웃 시 INTER_AREA), 상단 검은 스티칭
   경계 회피 top_margin=160px.

샘플(600~780s) 성능: 분석 22fps + 렌더 22fps. 유닛 테스트 6개
(`test_ptz_plan.py`, ultralytics 불필요) — 게이팅/보간/줌아웃/스무딩/미끼.

## 방향 결정 (사용자)

- 자동 공 인식은 신뢰 한계 → **GUI 에서 주기적으로 공 위치를 사용자가
  입력(키프레임)하는 방식**으로 간다. 자동 검출은 보조(후보 제시/사이
  구간 보간)로. 2패스 구조상 사용자 키프레임은 build_plan 의 고신뢰
  공 샘플로 그대로 끼워넣으면 됨.
- 향후 고급: **선수 움직임·킥 동작 기반 카메라 예측** (포즈/모션 큐).
- 향후: **선수 전원 개별 추적** (ultralytics track 모드 / ByteTrack 계열,
  ID 유지) — PitchAnalysis 계열 분석 기능의 기반.
