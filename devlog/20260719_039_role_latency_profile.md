- 종류: 성능/버그 수정
- 관련: [038](20260719_038_review_ui_batch.md)

## 프로파일 (합성 실경기 규모: 18k 샘플 × 20 검출, 트랙릿 2100)

| 단계 | 시간 |
|---|---|
| team_features (전 검출 스캔) — 분석당 1회 캐시 | 1018 ms |
| classify_teams (feats 캐시 사용) — 역할 지정마다 | 33 ms |
| **구 _ar_side × 선심 12조각 — 역할 지정마다** | **675 ms** |
| 신 _foot_med (분석당 1회 캐시) | 457 ms |
| 신 _ar_side × 12조각 (캐시 후) | 0.9 ms |
| 목록 그룹핑/정렬 | 4 ms |

잔여 지연의 주범은 **_ar_side** (선심 ARN/ARF 자동 판정): 선심 조각
1개당 전 분석 재스캔인데 `_ar_side_cache` 를 `_roles_changed` 가 매번
비워서, 선심 색 전파로 조각이 늘수록 역할 지정마다 수백 ms~수 초.

## 수정

- `_foot_med` — 트랙릿별 발 위치 중앙값을 분석당 1회 캐시,
  `_ar_side` 는 그 위에서 그룹 중앙값 변환 몇 번 (675ms → 0.9ms).
- 앞서 커밋한 `team_features` 캐시(8071877)·목록 색 벡터화(afa656a)와
  합쳐 역할 지정 경로의 파이썬 비용은 ~40ms. 남는 것은 Qt 목록 재구성
  (~2천 행, uniform size 적용) — 실기에서 여전히 느리면 다음 후보.

## 역할 지정 가림 버그 (팀 바꿔도 안 바뀜)

병합 그룹 멤버를 우클릭해 역할을 바꾸면 멤버 tid 에 저장되는데,
`_role_of` 는 대표 역할을 우선하므로 대표에 기존 역할이 있으면 새
지정이 가려졌다. `_set_role` 이 대표에 저장하고 멤버의 낡은 개별
역할을 지우도록 수정 (그룹 = 한 사람 원칙 일관화).

## 확인된 동작 (사용자 질문)

크롭 계획: 공 트랙 갭 ≤ gap_fill_sec(2s)는 공 위치 선형 보간만,
키프레임 간은 8s 까지 직접 브리지 — 선수 가중 목표·줌아웃은 2s 초과
공 부재 구간에만 발동. (질문대로 이미 그렇게 동작.)
