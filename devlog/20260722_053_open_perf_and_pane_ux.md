## 열기/트랙 로드 성능 — 세 겹의 캐시

- **선수 목록 O(n²) 제거**: 아이템 4천 개가 각각 색 캐시 함수를 호출,
  숨김 필터가 매번 전 트랙릿 재스캔 (_rep 839만 호출). 호이스팅으로
  열기 29.7s → 7.7s.
- **파생 요약 캐시** (.analysis.cache.json): 스팬·대표색·발 중앙값 —
  분석 (size, mtime) 키. 재열기 7.7s → **3.6s**. 헤드리스가 예열.
- **트랙 연결 캐시** (.analysis.link.cache.npz): link_ball_tracks 수십
  초 → 즉시 ("영상은 금방, 트랙이 느림" 보고의 원인). 역시 예열.
- **타이밍 사이드카** (<pano>.timing.json): 헤드리스 단계별 소요 기록
  — ocr_cached 41s vs 옛 시크 OCR ~13분 (P09-1 실측 ~19배).

## 페인 UX

- **줌/팬**: 휠 = 커서 기준 1~12×, 가운데 드래그 = 팬 (읽기 전용
  페인은 좌드래그), 가운데 더블클릭 = 리셋. 핵심 불변식: 방출 좌표는
  항상 전체 프레임 비율 — 편집 로직(키프레임·랜드마크·박스)이 줌을
  모른 채 정확히 동작 (계약 스모크).
- **레이더 = 위젯 오버레이**: 프레임 합성 → 페인 자식 위젯. 줌/팬과
  무관하게 우하단 고정, 마우스 투과, 불투명도 슬라이더 유지. 부모
  스타일시트 배경 상속으로 불투명해진 회귀 1건 수정 (transparent).
- **타이틀 칩**: 페인 좌상단에 영상 이름 + 현 시점 앵글 유무
  ("앵글: C0011" / "앵글 없음" / "이 시점 영상 없음").
