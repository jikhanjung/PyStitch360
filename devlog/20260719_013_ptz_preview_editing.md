## 분석 안정화 (완주 재분석 대비)

- **진행률 표시**: 분석 진행을 진행바에 퍼센트·fps·남은 시간으로 표시.
- **체크포인트/재개**: analyze_video 가 주기적으로 part.json (atomic) 저장,
  영상/파라미터/모델이 일치하면 이어서 재개 — 장시간 실행이 중단에도
  생존, 취소 시 그 프레임까지 저장. (트래커 ID는 경계에서 재시작하지만
  팀 분류가 트랙릿 단위라 영향 미미)
- **det_w 자동 스케일**: 기본값을 파노라마 폭의 절반(stride-32, 상한
  4416)으로 — 오늘 소스(5906px)는 기존과 동일한 2944, 8K급 소스는
  4416으로 자동 상향 (조용한 화질 저하 방지).

## 검수: 목록 → 미리보기 직접 조작으로

- 무시/복원 동작을 두 목록 사이 `>>` `<<` 버튼으로 이동 (Del/화살표 유지).
- **트랙 승격 (force_ranges)**: 사용자가 승격한 트랙은 그 시각·그 자리
  (f, x, y)를 지나는 한 static/isolated/overlap 자동 필터를 우회.
  무시가 승격보다 우선. (테스트 추가)
- **미리보기에서 바로 조작**: 프레임의 모든 공 후보를 상태색으로 표시
  (초록=채택, 자홍=승격, 빨간 X=무시, 회색=자동 기각). 좌클릭 = 상태별
  기본 동작 (회색/초록 → 경기 공으로 승격 + 경쟁 후보 트랙 자동 무시,
  자홍 → 승격 해제, 빨강 → 복원, 키프레임 → 삭제, 빈 곳 → 키프레임
  추가). 우클릭 = 전체 동작 메뉴. 호버 시 노란 링 (FramePane 에
  hover/context_requested 시그널 추가).

## 크롭 박스 = 키프레임 (그린 대로 나온다)

- 키프레임에 선택적 4번째 요소 **crop_w**: build_plan 이 그 샘플에서
  깊이 기반 줌을 오버라이드하고 위치+폭을 함께 앵커 (가우시안 윈도우,
  와이드 모드는 고정 폭 유지).
- PTZ 미리보기의 계획 크롭 박스가 **편집 가능**: 테두리 드래그 = 이동,
  모서리 핸들 = 중심 기준 16:9 고정 리사이즈(줌), 놓으면
  [frame, cx, cy, crop_w] 키프레임 커밋. 편집 중 노란 펜딩 박스,
  호버 피드백(굵은 테두리·커서 변경). 단순 클릭 키프레임은 기존
  3요소(자동 줌) 유지.

## 사이드카 정리 (2단계)

1. 기존 사이드카 2개를 `<pano>.ptz.json` 하나로 통합
   ({analysis, keyframes, ignores}, 구파일 자동 마이그레이션, atomic
   write, 편집 디바운스 저장).
2. 이후 다시 **분리**: 검수 마킹이 비파괴라 분석 블롭은 순수 기계
   출력 — `<pano>.analysis.json` (분석 완료 시 1회 기록, 검수가 건드리지
   않음) / `<pano>.ptz.json` ({keyframes, ignores, promotes} 만) 으로.
   편집 저장이 수 MB 재기록 → 수 KB. 구 통합 파일은 로드 시 자동 분리,
   외부의 더 새로운 .analysis.json 우선 규칙은 유지.
