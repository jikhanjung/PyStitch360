# P08: 경기 지표 — 점유율·패스맵 기본 구현 계획

- 날짜: 2026-07-21
- 종류: 계획
- 관련: [P03](20260719_P03_highlights_roadmap_plan.md) 4단계 (히트맵
  리포트 — 완료, core/report.py), [P06](20260720_P06_multicam_fusion_plan.md)
  7단계 (융합 후 정밀판의 자리)

## 배경 (사용자 방향, 2026-07-21)

선수 트래킹이 좋아지면 히트맵 — **이미 있음** (P03-4: 팀/선수 히트맵
PNG + 활동량 players.md). 공 트래킹이 완벽해지면 패스맵·점유율 —
완벽 전이라도 **기본 구현은 지금 가능**하다는 판단. 단, 현 트래킹
품질의 한계를 숨기지 않는 형태로:

- **관측된 샘플에서만 계산, 커버리지를 항상 병기** — 공백을 보간으로
  창작하지 않는다 (auto-level 042 교훈의 지표판).
- 모든 지표는 검수 결과(.analysis.json + .ptz.json + .events.json)의
  **파생물** — 원본 사이드카 불변, 리포트 산출물로만.

## 1단계: 소유(possession) 추정 코어

- `core/metrics.py` (신규): 샘플별 수락 공 위치 × 선수 필드 좌표 →
  최근접 선수 (거리 임계 ~2m, 지속 ~0.5s) → **소유 구간**
  [(t0, t1, tid, 팀)]. 두 팀이 동시에 근접(경합)이면 "불확실" 라벨.
- 점유율 = 팀별 소유 시간 비. 분모는 인플레이 시간 — match_info 의
  중단 구간(pauses)과 하프 경계(킥오프 앵커) 제외.
- **신뢰도 병기**: 공 관측률(샘플 커버리지), 경합 비율. 공백 구간은
  "미관측"으로 분리 집계 (어느 팀에도 배정하지 않음).
- 합성 테스트: 스크립트된 소유 시퀀스 주입 → 구간·점유율 복원.

## 2단계: 패스 추출 → 패스맵

- 소유 전이 분류: 같은 팀 다른 선수 = **패스**, 상대 팀 = 턴오버
  (인터셉트/태클), 아웃오브플레이 경계는 중단 구간과 대조.
- 킥 순간은 공 속도/방향 급변으로 스냅 (기존 궤적 물리 필터 재사용).
- **패스맵 PNG** (리포트 디렉터리에 추가): 선수 노드 = 평균 위치
  (히트맵과 동일 좌표계), 화살표 두께 = 횟수, OCR 등번호 라벨.
  팀별 1장 + 턴오버 맵 1장.
- 한계 명시: 파노라마 원경 공 공백 구간의 패스는 미검출 — 리포트에
  "미관측 소유 전이 n건" 으로 집계 (P06-4 융합이 개선 경로임을 문서화).

## 3단계: PitchWatch 통합 + 선수별 뛴 거리 승격

- 분석 메뉴 "경기 지표 (점유율·패스맵)" → 기존 리포트 생성기에 섹션
  추가 (players.md → match.md 로 확장: 점유율 표 + 패스맵 이미지 참조).
- **통계 결과 화면 분리 (사용자 지적, 2026-07-21)**: PtzTab 이
  PitchWatch 의 메인이므로 지표는 파일 산출물만으론 부족 — 별도
  **통계 창** (QDialog/보조 윈도우): 점유율·뛴 거리 표 위젯 + 히트맵/
  패스맵 이미지 페인, "리포트 폴더 열기" 버튼. 검수(PTZ)와 결과
  열람을 분리해 창 전환 없이 나란히 볼 수 있게. v2 에서 멀티캠
  match 레벨로 승격 시 이 창이 경기 대시보드가 된다.
- **선수별 뛴 거리/속도** (사용자 추가 요청): 계산은 이미 있음
  (core/report.py — 0.5s 창 리샘플로 지터 부풀림 방지, 11 m/s 상한
  게이트). match.md 에 팀별 정렬 표로 승격하되 **관측 시간 비율을
  같은 행에 병기** — 트랙릿 커버리지 60% 선수의 6km 와 95% 선수의
  6km 는 다른 숫자다. 병합·OCR 등번호 반영은 리포트 경로 그대로.
  원경 선수의 정확도 균일화는 4단계(융합)에서.
- 소유 구간을 타임라인 레인(팀 색 리본)으로 표시 — v1 은 표시만,
  검수(수정)는 다음 판.
- 턴오버 클러스터를 하이라이트 후보 신호에 공급 (P03 융합 스코어러에
  이벤트 소스 하나 추가).

## 4단계 (후속 — P06 연동)

- 융합 트랙(match 레이어)으로 같은 코드 재실행 — 원경 공백 해소 후
  정밀판. 입력만 바뀌고 지표 코드는 불변이 목표.
- 오프사이드 라인 검증 등 P06-7 잔여는 그때.

## 완료 기준

- 1: 합성 점유 시나리오 오차 < 2%p, 실경기(20260712) 점유율이
  하이라이트 육안 인상과 모순 없음 + 커버리지 수치 출력.
- 2: 합성 패스 시퀀스 완전 복원. 실경기 스팟체크 — 하이라이트 구간
  10개의 패스/턴오버 라벨 육안 일치율 보고.
- 3: 메뉴 1회 실행으로 match.md + 패스맵 PNG, 타임라인 소유 리본 표시.
